# AGENTS.md

## Назначение
S-Tracker - прототип системы управления задачами по бизнес-доменам.
Цель работы: сохранять рабочий прототип и постепенно улучшать структуру проекта.

## Главный принцип
Сначала работоспособность. Потом рефакторинг.
Любое изменение должно сохранять текущие пользовательские сценарии.

## Нельзя ломать

- открытие и сборку проекта;
- список задач и режим "Все задачи";
- переключение бизнес-доменов;
- статусы над таблицей;
- поиск и фильтры;
- настройки колонок;
- библиотеку атрибутов;
- пресеты колонок;
- связь данных, фильтров, колонок и таблицы.

## Источник истины
Данные: `src/data`.
Главная модель задач, доменов, очередей, атрибутов и значений: `src/data/task-model.js`.
Правила выборки, фильтрации, колонок и сборки строк таблицы: `src/domain`.
Поведение интерфейса: `src/features` и `src/ui`.
Внешний вид: `src/styles`.
UI не должен сам придумывать данные, атрибуты или доменную структуру.

## Документация
Общее понимание проекта: `PROJECT-OVERVIEW.md`.
Пользовательские пути: `docs/user-flows`.
Компоненты интерфейса и главный чертеж UI: `docs/components`.
Документация одна для команды и AI-агента: это конкретная спецификация интерфейса, а не абстрактное описание.
Пилот нового стандарта: `docs/components/filters`.
Стандарт оформления документации: `docs/documentation-standards.md`.
Вся документация в `docs/` должна соответствовать `docs/documentation-standards.md`.
Если меняется поведение интерфейса, обновить соответствующий flow.
Если меняется компонент, обновить его описание.

## Особенности проекта
Текущие бизнес-домены - набор для одной роли, а не полный список системы.
Разные роли могут видеть разное количество бизнес-доменов.
`dist` - результат сборки. Основные правки делать в `index.html` и `src`.

## Правило удаления
Удалять можно только то, что точно не участвует в работе прототипа.
Перед удалением проверить `index.html`, `src/main.js`, импорты, сборку и влияние на интерфейс.
Если есть сомнение - не удалять.
