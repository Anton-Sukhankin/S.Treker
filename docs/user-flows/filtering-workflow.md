# Flow: поиск, фильтры и пересборка списка задач

Этот flow связывает левую навигацию, очереди над списком задач, панель инструментов, панель фильтрации, список задач и пагинацию.

Детальное поведение отдельных элементов находится в компонентных спецификациях. Этот файл фиксирует только передачу состояния между компонентами.

## Старт сценария

1. В левой навигации выбран `Все задачи`, бизнес-домен, значение группировки или пользовательская группа.
2. В компоненте очередей активен один этап согласования.
3. Пользователь видит карточный или табличный список задач.
4. Пагинация показывает страницу текущего итогового набора, если в наборе больше 20 задач.

## Участники и роли

| Компонент | Роль в flow |
|---|---|
| Левая навигация | Передает выбранную область задач. |
| Очереди | Передают активный этап согласования. |
| Панель инструментов | Передает поисковую строку и открывает панель фильтрации. |
| Панель фильтрации | Передает примененные параметры фильтрации и счетчик активных параметров. |
| Список задач | Собирает итоговый набор задач и показывает его карточками или таблицей. |
| Пагинация | Делит итоговый набор на страницы и показывает задачи текущей страницы. |

## Основной путь

1. Пользователь выбирает область задач в левой навигации.
2. Заголовок правой рабочей области получает название выбранной области.
3. Пользователь выбирает этап согласования в сегмент-контроле очередей.
4. Список задач показывает задачи выбранной области и выбранного этапа.
5. Пользователь вводит текст в `Поиск по задачам`.
6. Список задач сразу показывает только задачи, где введенный текст найден в значениях задачи.
7. Пользователь открывает `Фильтры`, выбирает параметры и нажимает `Применить`.
8. Правая панель фильтрации закрывается.
9. Список задач показывает задачи, которые одновременно проходят выбранную область, активный этап, поиск и примененные параметры фильтрации.
10. Пагинация получает новый итоговый набор и возвращает текущую страницу к `1`.

## Передача состояния

| Источник | Условие | Получатель | Видимый итог |
|---|---|---|---|
| Левая навигация | Выбран `Все задачи`, бизнес-домен, значение группировки или пользовательская группа | Список задач | В списке остаются задачи выбранной области. |
| Очереди | Выбран этап согласования | Список задач | В списке остаются задачи активного этапа. |
| Панель инструментов | Поле `Поиск по задачам` заполнено | Список задач | В списке остаются задачи с совпадением по значениям задачи. |
| Панель фильтрации | Пользователь нажал `Применить` | Список задач | В списке остаются задачи, проходящие все активные параметры панели. |
| Панель фильтрации | Активен минимум один параметр | Кнопка `Фильтры` в панели инструментов | Иконка фильтра скрыта; круглый счетчик показывает количество активных параметров. |
| Список задач | Итоговый набор изменился после области, этапа, поиска или фильтров | Пагинация | Текущая страница получает значение `1`; страницы строятся по новому итоговому набору. |
| Список задач | Итоговый набор пуст | Пагинация и список задач | Пагинация скрыта; список задач показывает пустое состояние. |
| Список задач | Итоговый набор содержит 21 задачу или больше | Пагинация | Пагинация видима и показывает доступные номера страниц. |

## Когда обновлять этот документ

Обновлять, если меняется сквозная передача состояния между левой навигацией, очередями, панелью инструментов, фильтрами, списком задач или пагинацией.

Если меняется только один элемент внутри компонента, обновлять нужно компонентную спецификацию этого компонента, а этот flow менять только при изменении межкомпонентного результата.
