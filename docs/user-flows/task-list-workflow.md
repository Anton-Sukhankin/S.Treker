# Flow: работа со списком задач

Основной пользовательский путь списка задач теперь описан в спецификации компонента:

`docs/components/tasks/02-behavior.md`

Этот файл остается навигационной ссылкой, чтобы старые ссылки на flow не ломались.

## Где смотреть детали

| Что нужно понять | Где описано |
|---|---|
| Из каких частей состоит список задач | `docs/components/tasks/01-structure.md` |
| Основной пользовательский путь | `docs/components/tasks/02-behavior.md` |
| Данные, состояния, правила отбора и индикации | `docs/components/tasks/03-data-states-rules.md` |

## Правило обновления

Если меняется список задач, обновлять нужно спецификацию в `docs/components/tasks/`.

Этот файл не должен содержать отдельную версию сценария, чтобы не создавать вторую точку истины.
