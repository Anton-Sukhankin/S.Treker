# Пользовательские пути

В этой папке описаны основные сценарии работы пользователя.

Каждый файл отвечает на вопросы:

- с чего начинается сценарий;
- что делает пользователь;
- какие части интерфейса участвуют;
- какие состояния меняются;
- где нужно обновлять документацию при изменениях.

## Список flow

- `task-list-workflow.md` - основной путь работы со списком задач.
- `filtering-workflow.md` - применение поиска и фильтров.
- `column-settings-workflow.md` - настройка колонок и добавление атрибутов.
- `column-presets-workflow.md` - сохранение, выбор и удаление пресетов колонок.

Для компонентов, уже переведенных в новый формат, основной пользовательский путь хранится в `docs/components/<component>/02-behavior.md`.

Файлы в этой папке не должны создавать вторую версию сценария, если сценарий уже описан внутри спецификации компонента.

## Как добавлять новый flow

1. Скопировать структуру из `../templates/user-flow-template.md`.
2. Создать новый файл в этой папке.
3. Добавить ссылку в список выше.
4. Если flow зависит от конкретного компонента, обновить файл этого компонента в `../components/`.
