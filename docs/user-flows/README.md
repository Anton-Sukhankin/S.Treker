# Пользовательские пути

`docs/user-flows/` хранит сквозные сценарии, в которых участвуют несколько компонентов интерфейса.

Формат, обязательные разделы и правила написания flow зафиксированы в `docs/documentation-standards.md`, раздел `Сквозные пользовательские пути`.

## Когда использовать

| Ситуация | Где описывать |
|---|---|
| Сценарий проходит через несколько компонентов | Соответствующий файл в `docs/user-flows/`. |
| Сценарий полностью находится внутри одного компонента | `docs/components/<component>/02-behavior.md`. |
| Меняется правило состояния одного компонента | `docs/components/<component>/03-data-states-rules.md`. |

## Действующие flow

| Файл | Сценарий |
|---|---|
| `filtering-workflow.md` | Поиск, фильтры и пересборка списка задач через соседние компоненты. |
| `grouping-workflow.md` | Создание группировки из таблицы и выбор значения в левой навигации. |
| `bulk-actions-group-workflow.md` | Выбор задач, создание пользовательской группы и результат в левой навигации. |

Сценарии, которые полностью описываются внутри одного компонента, находятся в `docs/components/<component>/02-behavior.md` и не дублируются в этой папке.

## Как добавлять новый flow

1. Проверить, что сценарий проходит через несколько компонентов.
2. Скопировать структуру из `docs/templates/user-flow-template.md`.
3. Создать новый файл в этой папке.
4. Добавить файл в таблицу `Действующие flow`.
5. Обновить спецификации компонентов, если изменились их структура, поведение или правила состояния.
