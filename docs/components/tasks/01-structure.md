# Список задач: структура интерфейса

Файл описывает, где находится список задач и из каких видимых частей он состоит. Действия пользователя описаны в `02-behavior.md`; данные, состояния и правила отбора - в `03-data-states-rules.md`.

## 1. Границы компонента

Список задач находится в правой рабочей области, ниже заголовка, верхней панели инструментов и очередей.

| Входит в компонент | Не входит в компонент, но влияет на него |
|---|---|
| Карточный список задач | Левая навигация и бизнес-домены |
| Табличный список задач | Очереди над списком |
| Строка задачи в таблице | Глобальный поиск |
| Карточка задачи | Правая панель фильтрации |
| Чекбоксы выбора задач | Настройки колонок |
| Пагинация под списком | Кнопки скачать, печать, создать задачу |
| Пустое состояние списка | Системный overlay |

Компонент всегда показывает итоговый набор задач после применения контекста левой навигации, активной очереди, поиска, фильтров и текущей страницы пагинации.

## 2. Режимы отображения

В прототипе есть два режима списка.

| Режим | Контейнер | Когда виден | Что показывает |
|---|---|---|---|
| Карточный вид | `#view-mode-cards`, внутри `#js-main-task-list` | По умолчанию при открытии прототипа | Плоский список карточек задач |
| Табличный вид | `#view-mode-table`, внутри таблица с `#js-unified-table-body` | После выбора табличного режима в переключателе вида | Таблицу с динамическим набором колонок |

Переключатель вида находится в верхней панели списка задач. Сам переключатель является частью верхней панели, но его результат относится к списку: один режим скрывается, второй показывается.

## 3. Общие элементы карточки и строки

Карточка задачи и строка таблицы показывают один и тот же объект `Task`, но в разной компоновке.

| Общий элемент | Карточный вид | Табличный вид |
|---|---|---|
| Чекбокс выбора | Отдельный элемент слева от содержимого карточки | Ячейка первой закрепленной колонки `cb` |
| Номер задачи | В основной зоне карточки перед наименованием | Ячейка закрепленной колонки `id` |
| Наименование | В основной зоне карточки после номера | Ячейка закрепленной колонки `title` |
| Иконка задачи | Слева от текста наименования | Слева от текста наименования |
| Статус задачи | В правой зоне карточки | В ячейке колонки `status`, если колонка активна |
| Автор | В правой зоне карточки | В ячейке колонки `author`, если колонка активна |
| Дата | В правой зоне карточки | В ячейке соответствующей date-колонки |

Общие правила:

- наименование длиннее ширины контейнера обрезается многоточием;
- перед наименованием всегда стоит иконка задачи 16 x 16 px;
- перед датой всегда стоит иконка календаря 16 x 16 px;
- автор отображается через аватар 24 x 24 px, имя и email;
- email автора является кликабельным элементом копирования;
- статус отображается цветным тегом;
- состояния `hover`, `selected`, `is-new` и `is-overdue` применяются и к карточке, и к строке.

## 4. Карточка задачи

Карточка задачи - одна горизонтальная строка в карточном виде.

| Зона карточки | Что содержит |
|---|---|
| Левая зона | Чекбокс выбора задачи |
| Основная зона | Номер задачи и наименование задачи |
| Правая зона | Тег статуса задачи, автор, дата |

## 5. Состав карточки

| Элемент | Как отображается |
|---|---|
| Чекбокс | Квадратный контрол выбора задачи слева от содержимого |
| Номер | Значение `id`, например `ECM-1001` |
| Наименование | Иконка задачи 16 x 16 px, затем текст наименования; длинный текст обрезается многоточием |
| Статус | Цветной контрастный тег с текстом статуса |
| Автор | Аватар 24 x 24 px, имя автора, email под именем |
| Email автора | Кликабельный текст, по клику копирует адрес |
| Дата | Иконка календаря 16 x 16 px, затем значение даты |

В карточке показывается одна дата. Для обычной задачи это дата создания. Для просроченной задачи в домене `tracker` показывается срок, в домене `control` - дата проверки.

## 6. Табличный вид

Табличный вид состоит из контейнера горизонтального скролла, таблицы, заголовка таблицы и тела таблицы.

| Область | Что содержит |
|---|---|
| Заголовок таблицы | Названия активных колонок и служебные элементы заголовков |
| Тело таблицы | Строки задач текущей страницы |
| Контейнер таблицы | Горизонтальный скролл, если колонок больше ширины области |

Колонки таблицы собираются динамически. Состав зависит от выбранного пункта левой навигации и настроек колонок.

## 7. Закрепленные колонки таблицы

При горизонтальном скролле в таблице закреплены первые три колонки.

| Колонка | Ключ | Положение |
|---|---|---|
| Чекбокс | `cb` | Закреплена слева, первая колонка |
| Номер | `id` | Закреплена после чекбокса |
| Наименование | `title` | Закреплена после номера |

При горизонтальном скролле остальные колонки уходят под закрепленную область. У колонки `Наименование` появляется тень справа, если таблица проскроллена по горизонтали.

## 8. Заголовок таблицы

| Тип колонки | Что видно в заголовке |
|---|---|
| `cb` | Чекбокс выбора всех видимых строк таблицы |
| `id` | Текст `№` |
| `title` | Текст `Наименование` без подчеркивания |
| Группируемая колонка | Текст колонки и кнопка группировки справа |
| Обычная колонка | Только текст колонки |

Кнопка группировки не показывается в сгруппированном представлении и в пользовательской группе.

## 9. Строка таблицы

Каждая строка таблицы соответствует одной задаче текущей страницы.

Строка таблицы не является отдельным компонентом верхнего уровня. Это табличное представление той же задачи, которая в карточном режиме отображается как карточка.

| Тип ячейки | Как отображается значение |
|---|---|
| Чекбокс | Контрол выбора конкретной строки |
| Номер | `id` задачи |
| Наименование | Иконка задачи 16 x 16 px, затем подчеркнутый текст задачи; длинный текст обрезается многоточием |
| Статус | Цветной тег статуса |
| Автор | Аватар 24 x 24 px, имя автора, email под именем |
| Дата | Иконка календаря 16 x 16 px, затем значение даты |
| Система | Код бизнес-домена в верхнем регистре |
| Пустое значение | Длинное тире `–` |

## 10. Отличия карточки и строки таблицы

| Отличие | Карточка задачи | Строка таблицы |
|---|---|---|
| Количество видимых атрибутов | Показывает компактный набор: номер, наименование, статус, автор, дата | Показывает активный набор колонок таблицы |
| Управление колонками | Не зависит от настроек колонок | Полностью зависит от активных колонок |
| Горизонтальный скролл | Не используется | Используется, если колонок больше ширины области |
| Закрепленные области | Нет закрепленных зон | Закреплены `cb`, `id`, `title` |
| Выбор всех задач | Нет чекбокса выбора всех в карточном заголовке | Есть чекбокс выбора всех видимых строк в заголовке таблицы |

## 11. Пагинация

Пагинация находится под списком задач внутри правой рабочей области.

| Состояние | Что видно |
|---|---|
| Задач не больше размера страницы | Пагинация скрыта |
| Задач больше размера страницы | Показываются стрелки назад/вперед и номера страниц |
| Текущая страница | Номер страницы выделен |
| Первая страница | Стрелка назад disabled |
| Последняя страница | Стрелка вперед disabled |

Размер страницы в текущем прототипе - 20 задач.

## 12. Пустое состояние

Если после применения навигации, очереди, поиска, фильтров и пагинации нет задач для отображения, список показывает пустое состояние.

| Режим | Где показывается пустое состояние |
|---|---|
| Карточный вид | Внутри контейнера карточек |
| Табличный вид | В одной строке таблицы, ячейка растягивается на все активные колонки |

## 13. Связанные компоненты

| Компонент | Как связан со списком |
|---|---|
| Очереди | Определяют, задачи какой очереди попадут в список |
| Фильтрация | Уменьшает набор задач перед отображением |
| Настройки колонок | Меняют состав колонок табличного вида |
| Библиотека атрибутов | Добавляет в настройки колонок скрытые атрибуты, которые после применения могут появиться в таблице |
| Левая навигация | Ограничивает список всеми задачами, конкретным доменом, группой или сгруппированным значением |
| Массовые действия | Появляются после выбора задач чекбоксами |
