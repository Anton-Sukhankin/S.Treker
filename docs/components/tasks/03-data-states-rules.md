# Список задач: данные, состояния и правила

## 1. Владельцы состояния

| Область | Состояние или значение | Правило видимого результата |
|---|---|---|
| Список задач | Итоговый набор задач, режим отображения, видимые строки или карточки, визуальные состояния задач | Список показывает задачи после области, очереди, поиска, фильтров, сортировки и пагинации. |
| Панель инструментов | Поисковая строка и режим отображения | Список получает поисковую строку и выбранный режим отображения. |
| Левая навигация | Выбранная область задач | Список получает область задач: `Все задачи`, бизнес-домен, пользовательскую группу или значение группировки. |
| Очереди | Активный этап согласования | Список получает активную очередь для обычного списка задач. |
| Панель фильтрации | Примененные параметры фильтрации | Список получает примененные параметры и исключает задачи, которые их не проходят. |
| Пагинация | Текущая страница | Список показывает только задачи текущей страницы. |
| Массовые действия | Набор выбранных задач | Чекбоксы списка показывают выбранное состояние задач из набора массовых действий. |

## 2. Интерфейсные состояния списка

| Состояние | Значение | Видимый результат |
|---|---|---|
| Выбранная область задач | `Все задачи`, бизнес-домен, пользовательская группа или сгруппированное значение | Ограничивает набор задач до выбранной области. |
| Активная очередь | Один сегмент в компоненте `Очереди` | Ограничивает набор задач до активного этапа согласования. |
| Поисковая строка | Пустая или заполненная строка | Заполненная строка оставляет только задачи с совпадением по значениям задачи. |
| Примененные параметры фильтрации | Пустые или активные параметры | Активные параметры исключают задачи, которые не проходят условия фильтрации. |
| Режим отображения | Карточный или табличный | Один и тот же итоговый набор задач показан карточками или таблицей. |
| Примененные настройки колонок | Состояние колонок для табличного вида | Меняет только динамические колонки таблицы. |
| Текущая страница | Номер страницы | В списке видны только задачи текущей страницы. |
| Выбранные задачи | Набор отмеченных задач | Чекбоксы выбранных задач активны; массовая панель видима при непустом наборе. |

## 3. Правило сборки итогового набора

| Порядок | Правило | Результат |
|---:|---|---|
| 1 | Берутся задачи активной очереди | В наборе остаются задачи одного этапа согласования. |
| 2 | Применяется выбранная область задач | В наборе остаются задачи выбранного домена, группы, значения или всех доступных доменов. |
| 3 | Применяются поисковая строка и активные параметры фильтрации | В наборе остаются задачи, прошедшие поиск и фильтры. |
| 4 | Применяется сортировка | Новые задачи идут выше обычных; просроченные идут выше обычных после новых. |
| 5 | Применяется пагинация | В интерфейсе видны только задачи текущей страницы. |
| 6 | Применяется режим отображения | Итоговый набор показан карточками или строками таблицы. |

## 4. Правила выбранной области задач

| Выбранная область | Задача остается в наборе |
|---|---|
| `Все задачи` | Бизнес-домен задачи доступен текущей роли. |
| Конкретный бизнес-домен | Бизнес-домен задачи совпадает с выбранным доменом. |
| Сгруппированное значение | Задача принадлежит выбранному домену и имеет выбранное значение в атрибуте группировки. |
| `Без значения` внутри группировки | У задачи нет значения в атрибуте группировки. |
| Пользовательская группа | Задача входит в выбранную пользовательскую группу. |

## 5. Правила колонок таблицы

| Тип колонок | Правило |
|---|---|
| Закрепленные колонки | Чекбокс, `№` и `Наименование` всегда видимы, стоят первыми и не отключаются настройками колонок. |
| Динамические колонки | Состав и порядок зависят от выбранной области задач и примененных настроек колонок. |
| `Все задачи` | Таблица использует объединенный набор колонок доступных бизнес-доменов. |
| Конкретный бизнес-домен | Таблица использует колонки выбранного бизнес-домена. |
| Пользовательская группа | Таблица использует объединенный набор колонок, если группа не привязана к одному бизнес-домену. |
| Скрытая колонка | Колонка не попадает в таблицу после применения настроек колонок. |
| Атрибут из библиотеки | Колонка видна в таблице только после `Применить` в настройках колонок. |

Настройки колонок не влияют на карточный вид.

## 6. Правила значений задачи

| Значение | Правило отображения |
|---|---|
| Номер | Показан как номер задачи. |
| Наименование | Показано с иконкой задачи; длинный текст обрезается многоточием. |
| Статус | Показан как цветной тег с белым текстом. |
| Автор | Показан как аватар, имя и email. |
| Email автора | При клике копируется; выбор задачи не меняется. |
| Дата | Показана с иконкой календаря. |
| Значение отсутствует | В ячейке таблицы показан пустой маркер `–`. |
| Атрибут неприменим к домену задачи | В объединенной таблице показан пустой маркер `–`. |

## 7. Статусы задачи

Статус задачи является атрибутом задачи. Этапы над списком задач не являются статусами задачи.

| Статус | Системное значение | Цвет тега |
|---|---|---|
| Новая | `new` | Циан |
| Назначена | `assigned` | Фиолетовый |
| В работе | `in_progress` | Синий |
| Назначение снято | `assignment_removed` | Серый |
| Завершена | `completed` | Зеленый |
| Отменена | `cancelled` | Темно-серый |
| Отклонена | `rejected` | Красный |

Каждый статус отображается в виде контрастного цветного тега с белым текстом.

## 8. Визуальные состояния задачи

| Состояние задачи | Условие | Карточный вид | Табличный вид |
|---|---|---|---|
| Обычная | Нет признака новой задачи и нет признака просрочки | Белая карточка; при hover светло-серый фон | Белые ячейки; при hover светло-серый фон всех ячеек |
| Новая | У задачи есть признак новой задачи | Зеленоватая подложка и зеленая левая индикация | Зеленоватые ячейки и зеленая левая индикация в первой ячейке |
| Просроченная | У задачи есть признак просрочки | Розоватая подложка, красная левая индикация, красная дата просрочки | Розоватые ячейки, красная левая индикация в первой ячейке, красная дата просрочки |
| Выбранная | Задача входит в набор выбранных задач | Чекбокс активен; карточка сохраняет выбранное состояние до снятия выбора | Чекбокс активен; строка сохраняет выбранное состояние до снятия выбора |

У просроченной задачи красным выделяется значение даты, которая считается сроком или контрольной датой. Красная обводка у тега статуса не используется.

## 9. Правило даты просрочки

| Бизнес-домен | Дата, которая получает состояние просрочки |
|---|---|
| `Строительные договоры` | Дата создания |
| `Закупки ТМЦ` | Срок |
| `Поручения СД` | Дата проверки |
| `Поручения СД` в объединенном режиме с колонкой `Система` | Может подсвечиваться дата создания |

В карточном виде для просроченной задачи показана одна дата: срок для `Закупки ТМЦ`, дата проверки для `Поручения СД`, иначе дата создания.

## 10. Правила выбора задач

| Состояние выбора | Видимый результат |
|---|---|
| Набор выбранных задач пуст | Все чекбоксы пустые. |
| Выбрана одна задача | Чекбокс этой задачи активен. |
| Выбрано несколько задач | Чекбоксы выбранных задач активны. |
| Выбраны все видимые строки таблицы | Чекбокс в заголовке таблицы активен. |
| Выбрана часть видимых строк таблицы | Чекбокс в заголовке таблицы находится в состоянии indeterminate. |

Чекбокс в заголовке таблицы получает состояние только по видимым строкам текущей страницы.
Плавающая панель и групповые кнопки описаны в компоненте `Массовые действия`.

## 11. Контракты взаимодействия

| Связанный компонент | Роль связанного компонента | Условие | Результат в списке задач | Результат в связанном компоненте |
|---|---|---|---|---|
| Панель инструментов | Источник поисковой строки и режима отображения | Пользователь вводит поиск или переключает вид | Список получает поисковую строку или выбранный режим | Панель инструментов показывает введенный поиск и активный вариант переключателя. |
| Левая навигация | Источник выбранной области задач | Пользователь выбирает область задач | Список показывает задачи выбранной области | Активный пункт левой навигации получает выбранное состояние. |
| Очереди | Источник активного этапа согласования | Пользователь выбирает сегмент очереди | Список показывает задачи активной очереди | Выбранный сегмент получает активное состояние. |
| Панель фильтрации | Источник примененных параметров | Пользователь нажал `Применить` или быстро сбросил фильтры | Список показывает задачи, которые проходят примененные параметры | Счетчик фильтров и счетчики соседних областей получают значения по примененным параметрам. |
| Пагинация | Получатель итогового набора и источник текущей страницы | Итоговый набор задач собран | Список показывает задачи текущей страницы | Пагинация показывает страницы итогового набора. |
| Настройки колонок | Источник примененного набора колонок | Пользователь нажал `Применить` в настройках колонок | Табличный вид получает примененный набор динамических колонок | Панель настроек колонок закрыта или остается в примененном состоянии. |
| Левая навигация | Получатель результата создания группировки | Пользователь нажал действие группировки в заголовке колонки таблицы | Набор видимых задач, режим отображения и текущая страница не изменяются | Левая навигация показывает вложенную группировку под активным пунктом `Все задачи` или бизнес-доменом. |
| Левая навигация | Источник выбранного значения группировки | Пользователь выбрал значение группировки в левой навигации | Список показывает только задачи выбранного значения | Значение группировки получает активное состояние. |
| Массовые действия | Получатель набора выбранных задач | Пользователь выбирает чекбокс задачи или чекбокс заголовка таблицы | Чекбоксы выбранных задач активны | Плавающая панель массовых действий видна, если выбран минимум один чекбокс. |

## 12. Ограничения ответственности

| Область | Правило |
|---|---|
| Исходные данные задач | Список задач не изменяет значения задач. |
| Поисковая строка | Список задач не изменяет значение поля `Поиск по задачам`. |
| Параметры фильтрации | Список задач не изменяет примененные параметры фильтрации. |
| Бизнес-домены | Список задач не изменяет состав бизнес-доменов. |
| Очереди | Список задач не изменяет значение очереди у задачи. |
| Пресеты и библиотека атрибутов | Список задач не создает, не сохраняет и не удаляет пресеты или библиотечные атрибуты. |
