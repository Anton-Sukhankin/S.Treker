# Список задач: поведение

## 1. Основной пользовательский путь

1. Пользователь открывает прототип.
2. По умолчанию выбран раздел `Все задачи`.
3. По умолчанию выбран этап `К исполнению`.
4. По умолчанию активен карточный вид списка задач.
5. Список задач применяет выбранную область задач, выбранный этап, поиск, примененные фильтры и текущую страницу.
6. Пользователь видит итоговый набор задач в карточном виде.
7. Пользователь переключает список в табличный вид.
8. Список показывает тот же итоговый набор задач в таблице.
9. Пользователь взаимодействует с задачами: выбирает чекбоксы, копирует email автора, меняет страницу или возвращается в карточный вид.

## 2. Первичная загрузка

| Шаг | Состояние интерфейса |
|---|---|
| Открытие прототипа | Инициализировано состояние приложения |
| Область задач | Выбран раздел `Все задачи` |
| Этап согласования | Активен сегмент `К исполнению` |
| Вид списка | Активен карточный вид |
| Список задач | Показаны задачи выбранного этапа с учетом примененных фильтров, поиска, сортировки и текущей страницы |

## 3. Переключение карточного и табличного вида

| Действие пользователя | Состояние карточного вида | Состояние табличного вида | Состояние панели инструментов |
|---|---|---|---|
| Нажимает кнопку карточного вида | Карточный вид видим | Табличный вид скрыт | Кнопки `Настройка колонок`, `Скачать` и `Печать` скрыты |
| Нажимает кнопку табличного вида | Карточный вид скрыт | Табличный вид видим | Кнопки `Настройка колонок`, `Скачать` и `Печать` видимы |

Переключение вида не меняет выбранный раздел, выбранный этап, поиск, примененные фильтры, текущий набор задач и выбранные задачи.

## 4. Переключение этапа согласования

| Действие пользователя | Состояние сегмент-контролов | Состояние списка задач |
|---|---|---|
| Нажимает сегмент этапа | Нажатый сегмент становится активным | Список показывает задачи выбранного этапа |
| Этап изменен | Счетчики этапов остаются видимыми | Список применяет выбранный этап вместе с текущим разделом, поиском и фильтрами |
| Итоговый набор задач изменился | Пагинация возвращается к первой странице | Список показывает первую страницу нового набора |

Выбранный этап ограничивает список задач до одной очереди согласования.

## 5. Создание группировки из заголовка таблицы

| Действие пользователя | Исходное состояние | Состояние списка задач | Результат в левой навигации |
|---|---|---|---|
| Нажимает действие группировки в заголовке колонки | Активен табличный вид; выбран `Все задачи` или бизнес-домен; колонка поддерживает группировку | Список остается в табличном виде; текущий набор задач не меняется | Под текущим пунктом `Все задачи` или бизнес-доменом появляется или обновляется вложенная группировка `По ...`. |
| Нажимает действие группировки по тому же атрибуту повторно | В левой навигации уже есть группировка по этому атрибуту | Список остается в текущем состоянии | В левой навигации остается одна группировка по этому атрибуту с актуальным набором значений. |

Действие группировки не видно в карточном виде, в сгруппированном представлении и в пользовательской группе.

## 6. Изменение области задач

| Область слева | Состояние списка задач |
|---|---|
| `Все задачи` | Показаны задачи всех бизнес-доменов, доступных текущей роли |
| Конкретный бизнес-домен | Показаны только задачи выбранного бизнес-домена |
| Сгруппированное значение внутри бизнес-домена | Показаны только задачи выбранного бизнес-домена с выбранным значением группировки |
| Пользовательская группа | Показаны только задачи, входящие в выбранную пользовательскую группу |

При смене области задач список сохраняет активный этап и применяет его к новой области.

## 7. Поиск и фильтры

| Действие пользователя | Состояние списка задач |
|---|---|
| Вводит текст в поле `Поиск по задачам` | Список сразу показывает только задачи, совпадающие с введенным текстом |
| Очищает поле поиска | Список снова показывает задачи без поискового сужения |
| Нажимает `Применить` в панели фильтрации | Панель фильтрации закрывается, список показывает только задачи, прошедшие все выбранные параметры |
| Сбрасывает фильтры | Список показывает задачи текущей области и этапа без фильтрационного сужения |

Поиск применяется сразу. Фильтры изменяют список только после `Применить` в панели фильтрации или после быстрого сброса активных фильтров.

## 8. Выбор одной задачи

| Место действия | Состояние выбранных задач | Состояние элемента списка |
|---|---|---|
| Чекбокс карточки | Задача добавляется в набор выбранных задач или удаляется из него | Карточка получает или теряет выбранное состояние |
| Чекбокс строки таблицы | Задача добавляется в набор выбранных задач или удаляется из него | Строка получает или теряет выбранное состояние |
| Email автора | Набор выбранных задач не меняется | Email копируется в буфер обмена |

Выбор задачи одинаков для карточного и табличного вида: одна и та же задача имеет одно выбранное состояние независимо от текущего вида списка.

## 9. Поведение карточки задачи

| Действие или условие | Состояние карточки |
|---|---|
| Пользователь наводит курсор на обычную карточку | Фон карточки становится светло-серым, синяя обводка не появляется |
| Пользователь наводит курсор на наименование задачи | Текст наименования получает акцентный цвет |
| Пользователь выбирает карточку чекбоксом | Чекбокс становится активным, карточка получает выбранное состояние |
| Выбранная обычная карточка остается выбранной | Карточка сохраняет фон состояния наведения до снятия чекбокса |
| Карточка новой задачи | Карточка имеет зеленоватую подложку и зеленую левую индикацию |
| Карточка просроченной задачи | Карточка имеет розоватую подложку и красную левую индикацию; значение срока выделено красным |
| Пользователь нажимает email автора | Email копируется, выбор карточки не меняется |

## 10. Поведение строки таблицы

| Действие или условие | Состояние строки таблицы |
|---|---|
| Пользователь наводит курсор на обычную строку | Фон меняется у всех ячеек строки, включая закрепленные ячейки |
| Пользователь наводит курсор на наименование задачи | Текст наименования получает акцентный цвет |
| Пользователь выбирает строку чекбоксом | Чекбокс становится активным, строка получает выбранное состояние |
| Строка новой задачи | Все ячейки строки имеют зеленоватую подложку, первая ячейка показывает зеленую левую индикацию |
| Строка просроченной задачи | Все ячейки строки имеют розоватую подложку, первая ячейка показывает красную левую индикацию; значение срока выделено красным |
| Пользователь нажимает email автора | Email копируется, выбор строки не меняется |
| Пользователь скроллит таблицу горизонтально | Колонки `чекбокс выбора`, `номер`, `наименование` остаются закрепленными слева |

## 11. Общие правила карточки и строки

| Правило | Карточка | Строка таблицы |
|---|---|---|
| Использует одну и ту же задачу | Да | Да |
| Поддерживает выбор чекбоксом | Да | Да |
| Показывает email автора с копированием | Да | Да |
| Показывает статус цветным тегом | Да | Да, если колонка `Статус` активна |
| Показывает одну фиксированную дату | Да | Нет, даты зависят от активных колонок |
| Зависит от настроек колонок | Нет | Да, только в динамической части таблицы |
| Имеет выбор всех видимых задач | Нет | Да, через чекбокс заголовка |

## 12. Выбор всех видимых строк в таблице

Выбор всех через заголовок доступен только в табличном виде.

| Состояние видимых строк таблицы | Состояние чекбокса в заголовке |
|---|---|
| Не выбрана ни одна видимая строка | Пустой чекбокс |
| Выбрана часть видимых строк | Состояние `indeterminate`: внутри чекбокса синий квадрат 12 x 12 px по центру |
| Выбраны все видимые строки | Чекбокс с синей заливкой и белой галочкой |

| Действие пользователя | Состояние строк таблицы |
|---|---|
| Нажимает пустой чекбокс в заголовке | Все строки текущей видимой страницы становятся выбранными |
| Нажимает чекбокс в заголовке, когда все видимые строки выбраны | Со всех строк текущей видимой страницы снимается выбор |
| Снимает выбор с одной строки после выбора всех | Чекбокс в заголовке получает состояние `indeterminate` |

## 13. Связь с массовыми действиями

| Условие в списке задач | Результат для соседнего компонента массовых действий |
|---|---|
| Выбрана минимум одна задача | Плавающая панель массовых действий видна внизу экрана. |
| Количество выбранных задач изменилось | Счетчик плавающей панели получает количество выбранных задач. |
| Снято выделение со всех задач | Плавающая панель массовых действий скрыта. |

Полное поведение плавающей панели, групповых кнопок и модального окна перемещения описывает компонент `Массовые действия`.

## 14. Копирование email автора

| Действие | Состояние интерфейса |
|---|---|
| Пользователь нажимает email автора в карточке или таблице | Адрес копируется в буфер обмена |
| Копирование успешно | Появляется toast `Адрес скопирован` |
| Копирование не удалось | Появляется toast `Не удалось скопировать адрес` |

Клик по email не выбирает строку или карточку задачи.

## 15. Пагинация

| Действие пользователя | Состояние списка |
|---|---|
| Нажимает номер страницы | Список показывает задачи выбранной страницы |
| Нажимает кнопку навигации со стрелкой вперед | Текущая страница увеличивается на 1 |
| Нажимает кнопку навигации со стрелкой назад | Текущая страница уменьшается на 1 |
| Меняет этап, область задач или фильтры | Список пересобирается, страница возвращается к первой |

Детальные состояния стрелок, номеров страниц, скрытия пагинации и разрывов `...` описывает компонент `Пагинация`.

## 16. Горизонтальный скролл таблицы

| Действие | Состояние таблицы |
|---|---|
| Пользователь скроллит таблицу вправо | Колонки `чекбокс выбора`, `номер`, `наименование` остаются закрепленными слева |
| Таблица имеет горизонтальный сдвиг больше 0 | У закрепленной области видимо правое визуальное отделение |
| Пользователь возвращает скролл к началу | Правое визуальное отделение закрепленной области скрывается |

## 17. Пустой результат

| Условие | Состояние списка |
|---|---|
| После всех правил отбора нет задач | Вместо строк или карточек показано пустое состояние |
| Пользователь меняет область задач, этап, поиск или фильтры | Список заново применяет правила отбора и показывает новый результат |
