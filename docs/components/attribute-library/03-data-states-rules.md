# Библиотека атрибутов: данные, состояния и правила

## 1. Интерфейсные состояния библиотеки

| Состояние | Пустое или базовое значение | Активное значение | Видимый результат |
|---|---|---|---|
| Раскрытие библиотеки | Библиотека закрыта | Библиотека открыта | При открытом состоянии видна область библиотеки рядом с `Общие атрибуты`. |
| Поиск по библиотеке | Поле поиска пустое | Введен текст | Список атрибутов ограничен совпадениями по названию. |
| Доступность атрибута | Атрибут недоступен | Атрибут доступен | Доступный атрибут виден в списке библиотеки. |
| Наличие атрибута в черновике | Атрибута нет в черновике | Атрибут добавлен в черновик | Добавленный атрибут не виден в библиотеке. |
| Библиотечные добавления | Добавлений нет | Добавлен минимум один атрибут из библиотеки | Кнопка `Сбросить` в футере библиотеки активна. |
| Примененное состояние колонок | Таблица использует прежние колонки | Нажато `Применить` в настройках колонок | Таблица показывает колонки из примененного состояния. |

## 2. Правило доступности атрибута

Атрибут виден в библиотеке, если одновременно выполнены все условия.

| Условие | Требование |
|---|---|
| Скрыт по умолчанию | Атрибут не входит в дефолтный набор колонок таблицы. |
| Относится к текущему контексту | Атрибут принадлежит выбранному бизнес-домену или одному из доменов режима `Все задачи`. |
| Отсутствует в черновике | Атрибут еще не добавлен в список `Общие атрибуты`. |
| Проходит поиск | Название атрибута содержит введенный текст или поле поиска пустое. |

Если хотя бы одно условие не выполнено, атрибут не виден в списке библиотеки.

## 3. Правила выбранной области задач

| Выбранная область | Доступные атрибуты библиотеки |
|---|---|
| `Все задачи` | Скрытые атрибуты всех бизнес-доменов, доступных текущей роли. |
| Конкретный бизнес-домен | Скрытые атрибуты выбранного бизнес-домена. |
| Пользовательская группа | Скрытые атрибуты всех доступных бизнес-доменов. |
| Сгруппированное представление | Скрытые атрибуты всех доступных бизнес-доменов. |

Если один и тот же атрибут доступен в нескольких доменах, в библиотеке он показан один раз.

## 4. Правила черновика

| Состояние черновика | Видимый результат |
|---|---|
| Атрибут добавлен из библиотеки | В `Общие атрибуты` видна новая включенная колонка; в библиотеке этот атрибут не виден. |
| Атрибут возвращен в библиотеку | В `Общие атрибуты` строка атрибута не видна; в библиотеке атрибут снова доступен. |
| Выполнен сброс библиотечных добавлений | Все атрибуты, добавленные из библиотеки, удалены из черновика; обычные колонки остаются в черновике. |
| Библиотечных добавлений нет | Кнопка `Сбросить` неактивна. |
| Есть минимум одно библиотечное добавление | Кнопка `Сбросить` активна. |

Сброс библиотечных добавлений не равен действию `По умолчанию`. `Сбросить` удаляет только добавления из библиотеки; `По умолчанию` возвращает весь черновик колонок к базовому отображению.

## 5. Правило применения к таблице

| Состояние до `Применить` | Состояние после `Применить` |
|---|---|
| Атрибут добавлен из библиотеки и виден только в `Общие атрибуты` | В таблице видна новая колонка. |
| Атрибут возвращен в библиотеку и не виден в `Общие атрибуты` | Колонка не видна в таблице. |
| Библиотечные добавления сброшены в черновике | Таблица показывает колонки без библиотечных добавлений. |
| Панель закрыта без `Применить` | Таблица сохраняет прежний набор колонок. |

Библиотека не изменяет значения задач. Она меняет только набор колонок, который можно применить к табличному виду.

## 6. Пустые значения в таблице

| Условие | Видимый результат |
|---|---|
| У задачи есть значение добавленного атрибута | В ячейке показано значение атрибута. |
| У задачи нет значения добавленного атрибута | В ячейке показано пустое значение. |
| В режиме `Все задачи` атрибут относится не ко всем доменам | У задач без такого атрибута показано пустое значение. |

Пустое значение после добавления атрибута является допустимым состоянием объединенной таблицы.
