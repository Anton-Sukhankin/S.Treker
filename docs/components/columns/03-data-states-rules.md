# Настройки колонок: данные, состояния и правила

Файл описывает, какие данные и состояния нужны интерфейсу настройки колонок, как формируется черновик и по каким правилам изменения попадают в таблицу. Структура описана в `01-structure.md`, действия пользователя - в `02-behavior.md`.

## 1. Источники данных и зоны реализации

Этот раздел описывает не пути текущего проекта, а зоны кода, которые должны быть реализованы или подключены, чтобы настройки колонок работали.

| Что нужно настройкам | Что должно быть реализовано в коде | Как компонент это использует |
|---|---|---|
| Описание атрибутов | Список доступных атрибутов с ключом, названием, типом значения и служебными признаками | Из атрибутов строятся названия колонок и библиотека |
| Привязка атрибутов к бизнес-доменам | Правило, какие атрибуты относятся к каждому бизнес-домену | Определяет состав колонок для домена |
| Базовые колонки | Правило получения дефолтных колонок текущего контекста | Создает состояние `Базовое отображение` |
| Объединенные колонки для `Все задачи` | Правило объединения дефолтных колонок доступных доменов | Позволяет показать общую таблицу по всем доменам |
| Состояние активных колонок | Хранилище примененных пользовательских настроек по контексту | Определяет, какие колонки видны в основной таблице |
| Черновик настроек | Временное состояние панели до `Применить` | Позволяет менять панель без немедленного изменения таблицы |
| Библиотека атрибутов | Список скрытых атрибутов, доступных текущему контексту | Позволяет добавить скрытый атрибут в черновик колонок |
| Пресеты колонок | Хранилище сохраненных вариантов черновика по контексту | Позволяет быстро восстановить набор видимости и порядка колонок |
| Рендер таблицы | Функция перерисовки списка задач | После `Применить` показывает таблицу с новым набором колонок |

## 2. Основные сущности состояния

| Состояние | Назначение |
|---|---|
| Примененное состояние колонок | То, что уже влияет на основную таблицу |
| Черновик колонок | То, что пользователь меняет внутри открытой панели |
| Активный пресет | Название пресета, выбранного для текущего контекста |
| Черновик активного пресета | Название пресета внутри открытой панели до применения |
| Список пресетов | Сохраненные варианты состояния колонок по контексту |
| Контекст навигации | Раздел, для которого настраиваются колонки: `Все задачи`, бизнес-домен, группа или другое представление |

Главное правило: таблица читает примененное состояние, а панель редактирует черновик.

## 3. Состояние колонки

Состояние одной настраиваемой колонки хранит минимум два значения.

| Поле | Что означает |
|---|---|
| `key` | Ключ атрибута или колонки |
| `visible` | Показывать колонку после применения или скрыть |

Порядок колонок определяется порядком элементов в массиве состояния.

Фиксированные колонки `cb`, `id`, `title` не сохраняются в пользовательском состоянии. Они всегда добавляются таблицей отдельно и стоят первыми.

## 4. Контекст настройки

Настройки колонок хранятся отдельно для каждого контекста.

| Контекст | Какой набор колонок используется как база |
|---|---|
| `Все задачи` | Объединенный набор дефолтных колонок всех доступных бизнес-доменов |
| Конкретный бизнес-домен | Дефолтные колонки выбранного бизнес-домена |
| Пользовательская группа или другой id вне списка доменов | Объединенный набор колонок, как для `Все задачи` |

Один и тот же пресет или порядок колонок не переносится автоматически между разными контекстами.

## 5. Базовые, примененные и черновые колонки

| Тип состояния | Когда используется | Что содержит |
|---|---|---|
| Базовое состояние | Если пользователь ничего не настраивал или нажал `По умолчанию` | Все настраиваемые дефолтные колонки текущего контекста с `visible: true` |
| Примененное состояние | При рендере основной таблицы | Последнее состояние, сохраненное кнопкой `Применить` |
| Черновик | Пока открыта панель | Копию примененного состояния или базу, если примененного состояния нет |

Черновик создается при открытии панели. Если пользователь закрывает панель без применения, следующий запуск снова создаст черновик из примененного состояния.

## 6. Нормализация состояния

Перед сохранением состояние колонок очищается.

| Проверка | Что происходит |
|---|---|
| Нет массива состояния | Используется базовое состояние |
| Нет `key` или `key` не строка | Элемент отбрасывается |
| `key` относится к фиксированным колонкам `cb`, `id`, `title` | Элемент отбрасывается |
| `key` не входит в доступные колонки текущего контекста | Элемент отбрасывается |
| `key` повторяется | Повтор отбрасывается |
| `visible` не равен `false` | Колонка считается видимой |

Если после восстановления примененное состояние становится пустым, оно удаляется, и таблица возвращается к базовому набору.

## 7. Правило получения активных колонок таблицы

| Шаг | Правило |
|---|---|
| 1 | Берется базовый набор колонок текущего контекста |
| 2 | К нему добавляются колонки из примененного состояния, если они были добавлены из библиотеки |
| 3 | Фиксированные колонки `cb`, `id`, `title` ставятся первыми |
| 4 | Остальные колонки сортируются по порядку в примененном состоянии |
| 5 | Колонки с `visible: false` не попадают в таблицу |
| 6 | Таблица получает итоговый массив колонок |

## 8. Правила списка в панели

| Правило | Что видит пользователь |
|---|---|
| Фиксированные колонки исключены | В панели нет чекбоксов для `cb`, `id`, `title` |
| Колонка есть в черновике и `visible: true` | Чекбокс включен |
| Колонка есть в черновике и `visible: false` | Чекбокс выключен |
| Колонка добавлена из библиотеки | В строке есть действие возврата в библиотеку |
| Колонка не добавлена из библиотеки | Действия возврата нет |

Панель всегда отображает черновик, а не напрямую состояние основной таблицы.

## 9. Правила библиотеки в контексте настроек колонок

| Ситуация | Правило |
|---|---|
| Библиотека закрыта | Основная панель работает без левой области библиотеки |
| Библиотека открыта | Показываются атрибуты текущего контекста, которых нет в черновике |
| Пользователь добавил атрибут | Атрибут появляется в черновике с `visible: true` |
| Пользователь вернул атрибут в библиотеку | Атрибут удаляется из черновика |
| Пользователь нажал `Сбросить` в футере библиотеки | Все библиотечные атрибуты удаляются из черновика |

Добавление, возврат и сброс библиотечных атрибутов не меняют основную таблицу до `Применить`.

## 10. Правила кнопки `Сохранить`

| Условие | Состояние кнопки |
|---|---|
| Черновик совпадает с выбранным пресетом или базой | `Сохранить` disabled |
| Черновик отличается от выбранного пресета или базы | `Сохранить` active |

Сравнение выполняется по набору колонок, их порядку и `visible`-состоянию.

## 11. Правила кнопки `Применить`

При `Применить` выполняется цепочка:

| Порядок | Действие |
|---|---|
| 1 | Считывается текущий порядок и чекбоксы из списка колонок |
| 2 | Черновик нормализуется |
| 3 | Нормализованный черновик записывается как примененное состояние текущего контекста |
| 4 | Активный пресет текущего контекста обновляется значением из черновика |
| 5 | Панель и overlay закрываются |
| 6 | Библиотека закрывается |
| 7 | Таблица задач перерисовывается |
| 8 | Счетчики левой панели обновляются |

## 12. Что не меняют настройки колонок

| Область | Правило |
|---|---|
| Данные задач | Не меняются |
| Значения атрибутов | Не удаляются и не редактируются |
| Карточный вид | Не зависит от настроек колонок |
| Очереди | Не меняются |
| Фильтры | Не меняются |
| Состав бизнес-доменов | Не меняется |

Настройки колонок меняют только способ отображения атрибутов в таблице.
