# Левая навигация: поведение

## 1. Основной пользовательский путь

1. Пользователь открывает прототип.
2. В левой навигации активен пункт `Все задачи`.
3. Заголовок правой рабочей области получает текст `Все задачи`.
4. Список задач показывает задачи всех бизнес-доменов, доступных текущей роли, с учетом активного этапа над списком.
5. Пользователь кликает по пункту бизнес-домена.
6. Выбранный пункт бизнес-домена получает активное состояние; пункт `Все задачи` теряет активное состояние.
7. Заголовок правой рабочей области получает название выбранного бизнес-домена.
8. Список задач показывает задачи выбранного бизнес-домена с учетом активного этапа, поиска и примененных фильтров.
9. Счетчики в левой навигации получают значения по текущим данным и примененным фильтрам.

## 2. Выбор основной области задач

### 2.1 Назначение

| Объект выбора | Что получает пользователь |
|---|---|
| `Все задачи` | Правая рабочая область показывает задачи всех бизнес-доменов, доступных текущей роли. |
| Бизнес-домен | Правая рабочая область показывает задачи только выбранного бизнес-домена. |
| Активная область задач | Список задач, заголовок правой рабочей области и счетчики левой навигации получают один общий выбранный источник задач. |

### 2.2 Мини-путь пользователя

1. Пользователь видит пункт `Все задачи` и список бизнес-доменов в левой навигации.
2. Пользователь кликает по пункту `Все задачи` или по бизнес-домену.
3. Выбранный пункт получает активное состояние.
4. Остальные пункты основной области задач теряют активное состояние.
5. Заголовок правой рабочей области получает название выбранного пункта.
6. Список задач показывает задачи выбранной области с учетом активного этапа, поиска и примененных фильтров.

### 2.3 Реакции интерфейса

| Действие пользователя | Исходное состояние | Итог интерфейса |
|---|---|---|
| Клик по `Все задачи` | Активен бизнес-домен, значение группировки или пользовательская группа | `Все задачи` получает активное состояние; остальные пункты левой навигации теряют активное состояние; заголовок правой рабочей области получает текст `Все задачи`; список задач показывает задачи всех бизнес-доменов, доступных текущей роли, с учетом активного этапа, поиска и примененных фильтров. |
| Клик по бизнес-домену | Бизнес-домен видим в левой навигации и не активен | Выбранный бизнес-домен получает активное состояние; остальные пункты левой навигации теряют активное состояние; заголовок правой рабочей области получает название бизнес-домена; список задач показывает задачи выбранного бизнес-домена с учетом активного этапа, поиска и примененных фильтров. |
| Клик по активному пункту | Пункт `Все задачи` или бизнес-домен уже активен | Активный пункт сохраняет активное состояние; заголовок правой рабочей области сохраняет текущий текст; список задач сохраняет текущий набор задач. |

### 2.4 Правила области

| Правило | Значение |
|---|---|
| `Все задачи` | Не является бизнес-доменом; объединяет задачи всех бизнес-доменов, доступных текущей роли. |
| Бизнес-домены | Имеют одинаковую механику выбора; отличаются набором задач, названием в заголовке правой рабочей области и значениями счетчиков. |
| Активный пункт | В основной области задач активен только один пункт: `Все задачи`, один бизнес-домен, значение группировки или пользовательская группа. |
| Поиск, фильтры и этап | Не сбрасываются при выборе `Все задачи` или бизнес-домена; список задач применяет их к новому выбранному источнику задач. |

## 3. Создание группировки

### 3.1 Назначение

| Объект | Что получает пользователь |
|---|---|
| Действие группировки в заголовке колонки таблицы | Возможность разделить текущую область задач по значениям выбранного атрибута. |
| Вложенная группировка в левой навигации | Список значений выбранного атрибута под пунктом `Все задачи` или бизнес-доменом. |
| Значение группировки | Отдельную область задач, которую можно выбрать в левой навигации. |

### 3.2 Мини-путь пользователя

1. Пользователь находится в табличном виде списка задач.
2. Пользователь видит действие группировки в заголовке группируемой колонки.
3. Пользователь кликает по действию группировки.
4. Левая навигация показывает вложенную группировку под текущим пунктом `Все задачи` или бизнес-доменом.
5. Группировка получает заголовок `По ...` с названием выбранного атрибута.
6. Список значений группировки показывает значения выбранного атрибута и счетчики задач.
7. Пользователь выбирает значение группировки в левой навигации.

### 3.3 Реакции интерфейса

| Действие пользователя | Исходное состояние | Итог интерфейса |
|---|---|---|
| Клик по действию группировки в заголовке колонки таблицы | Активен `Все задачи` или бизнес-домен; табличный вид открыт; колонка поддерживает группировку | Под активным пунктом левой навигации виден блок вложенной группировки; заголовок блока получает формат `По ...`; список значений показывает значения выбранного атрибута. |
| Клик по действию группировки по тому же атрибуту | Под активным пунктом уже есть группировка по этому атрибуту | В левой навигации остается одна группировка по этому атрибуту; список значений получает актуальный набор значений. |
| Клик по действию группировки | Активно значение группировки или пользовательская группа | Действие группировки в заголовках колонок не видно; новая группировка не создается. |

### 3.4 Контракт компонентов

| Связанный компонент | Роль связанного компонента | Условие | Результат в левой навигации | Результат в связанном компоненте |
|---|---|---|---|---|
| Табличный список задач | Содержит действие группировки в заголовке группируемой колонки | Пользователь кликает по действию группировки | Под текущим пунктом появляется или обновляется вложенная группировка | Таблица сохраняет текущий набор задач и режим отображения. |
| Список задач | Получает выбранное значение группировки как условие отбора | Пользователь кликает по значению группировки в левой навигации | Значение получает активное состояние | Список показывает задачи выбранного значения. |

## 4. Выбор значения группировки

| Действие пользователя | Условие | Итог интерфейса |
|---|---|---|
| Клик по значению группировки | Значение видимо под пунктом `Все задачи` или бизнес-доменом | Значение получает активное состояние; основной пункт и остальные значения теряют активное состояние; заголовок правой рабочей области получает формат `Область • Значение`; список задач показывает задачи выбранного значения. |
| Клик по значению `Без значения` | Значение видимо | Список задач показывает задачи выбранной области, у которых атрибут группировки пустой. |
| Клик по значению группировки | Активны поиск или фильтры | Список задач показывает задачи выбранного значения, которые проходят текущий поиск и примененные фильтры. |

## 5. Раскрытие и сворачивание группировок

| Действие пользователя | Условие | Итог интерфейса |
|---|---|---|
| Клик по кнопке раскрытия у пункта `Все задачи` или бизнес-домена | У пункта есть созданные группировки; группировки скрыты | Область вложенных группировок под пунктом видима. |
| Клик по кнопке раскрытия у пункта `Все задачи` или бизнес-домена | У пункта есть созданные группировки; группировки видимы | Область вложенных группировок под пунктом скрыта. |
| Клик по заголовку группировки `По ...` | Список значений скрыт | Список значений внутри этой группировки видим. |
| Клик по заголовку группировки `По ...` | Список значений видим | Список значений внутри этой группировки скрыт. |

При сворачивании группировки выбранная область задач сохраняет текущее значение; список задач сохраняет текущий набор задач.

## 6. Сброс и удаление группировок

| Действие пользователя | Условие | Итог интерфейса |
|---|---|---|
| Клик по кнопке сброса группировок у пункта | У пункта есть минимум одна группировка | Все группировки этого пункта удалены из левой навигации. |
| Клик по кнопке сброса группировок у пункта | Активно значение группировки этого пункта | Активная область получает значение исходного пункта `Все задачи` или бизнес-домена; заголовок получает название исходного пункта; список задач показывает задачи исходного пункта. |
| Клик по кнопке удаления значения группировки | Значение видимо в списке группировки | Значение удалено из списка группировки. |
| Клик по кнопке удаления значения группировки | Удаляемое значение было активным | Активная область получает значение исходного пункта `Все задачи` или бизнес-домена; заголовок получает название исходного пункта; список задач показывает задачи исходного пункта. |
| Удаление последнего значения группировки | В группировке не остается значений | Группировка исчезает из левой навигации. |

## 7. Пользовательские группы

| Действие пользователя | Условие | Итог интерфейса |
|---|---|---|
| Создание группы из выбранных задач | Пользователь выбрал минимум одну задачу в списке и подтвердил создание группы | В левой навигации виден блок пользовательских групп; внутри блока виден новый пункт группы. |
| Клик по пользовательской группе | Группа видима | Группа получает активное состояние; заголовок правой рабочей области получает название группы; список задач показывает задачи этой группы. |
| Клик по кнопке удаления пользовательской группы | Группа видима | Группа удалена из левой навигации. |
| Клик по кнопке удаления пользовательской группы | Удаляемая группа была активной | Активная область получает значение `Все задачи`; заголовок получает текст `Все задачи`; список задач показывает задачи всех доступных бизнес-доменов. |
| Удаление всех задач из активной пользовательской группы | После удаления в группе не остается задач | Группа удалена; активная область получает значение `Все задачи`; список задач показывает задачи всех доступных бизнес-доменов. |

## 8. Закладки

| Действие пользователя | Условие | Итог интерфейса |
|---|---|---|
| Клик по заголовку закладки | Закладка свернута | Тело закладки с группами видимо. |
| Клик по заголовку закладки | Закладка раскрыта | Тело закладки с группами скрыто. |
| Клик по группе внутри закладки | Группа видима | Группа получает активное состояние; заголовок правой рабочей области получает название группы; список задач показывает задачи этой группы. |
| Клик по кнопке удаления закладки | Закладка видима | Закладка и все группы внутри нее удалены из левой навигации. |
| Клик по кнопке удаления закладки | Активна группа внутри удаляемой закладки | Активная область получает значение `Все задачи`; заголовок получает текст `Все задачи`; список задач показывает задачи всех доступных бизнес-доменов. |

## 9. Изменение ширины

| Действие пользователя | Условие | Итог интерфейса |
|---|---|---|
| Наведение на область между левой навигацией и правой рабочей областью | Курсор находится на области изменения ширины | Курсор получает состояние изменения горизонтального размера. |
| Зажатие левой кнопки мыши на области изменения ширины | Курсор находится на области изменения ширины | Начинается изменение ширины левой навигации. |
| Перетаскивание вправо | Изменение ширины активно; новая ширина меньше максимальной | Левая навигация получает большую ширину; правая рабочая область начинается правее. |
| Перетаскивание влево | Изменение ширины активно; новая ширина больше минимальной | Левая навигация получает меньшую ширину; правая рабочая область начинается левее. |
| Перетаскивание влево | Левая навигация достигла минимальной ширины | Ширина левой навигации сохраняет минимальное значение; правая рабочая область не заходит под левую навигацию. |
| Отпускание левой кнопки мыши | Изменение ширины активно | Изменение ширины завершается; курсор возвращается в обычное состояние вне области изменения ширины. |
