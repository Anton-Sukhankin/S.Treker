# Массовые действия: структура интерфейса

## 1. Граница функционального блока

`Массовые действия` — функциональный блок для действий над выбранными задачами. Главный UI блока — плавающая панель массовых действий. Чекбоксы выбора расположены в списке задач, но не являются частью плавающей панели. Выбор минимум одной задачи делает плавающую панель видимой.

| Объект | Роль в функциональности | Где расположен | Правило |
|---|---|---|---|
| Чекбокс карточки задачи | Элемент выбора одной задачи | Карточка задачи | При выборе чекбокса карточка входит в набор выбранных задач. |
| Чекбокс строки таблицы | Элемент выбора одной задачи | Первая закрепленная колонка строки таблицы | При выборе чекбокса строка входит в набор выбранных задач. |
| Чекбокс заголовка таблицы | Элемент выбора всех видимых строк | Первая закрепленная колонка заголовка таблицы | При выборе чекбокса все видимые строки текущей страницы входят в набор выбранных задач. |
| Выбранное состояние карточки или строки | Состояние элемента списка задач | Карточка или строка таблицы | Внешний визуальный признак задачи, включенной в набор выбора. |
| Плавающая панель массовых действий | Главный UI функционального блока | Нижняя центральная часть блока списка задач в правой рабочей области | Видна, если выбрана минимум одна задача. |
| Счетчик выбранных задач | Элемент плавающей панели | Плавающая панель | Показывает количество выбранных задач. |
| Кнопки массовых действий | Элементы плавающей панели | Плавающая панель | При клике выполняют действие над набором выбранных задач. |
| Модальное окно `Перемещение в группу` | Дочерний UI функционального блока | Поверх интерфейса | Слой ввода названия новой пользовательской группы. |
| Левая навигация | Внешний получатель результата | Левая панель | Показывает созданную пользовательскую группу или убирает пустую активную группу. |
| Toast-уведомление | Внешний получатель результата | Системная область уведомлений | Показывает результат создания группы или удаления задач из группы. |
| Список задач | Внешний получатель результата | Правая рабочая область | Снимает выбранное состояние задач после выполненного группового действия. |

## 2. Элементы выбора задач

| Элемент выбора | Условие видимости | Роль для функционального блока |
|---|---|---|
| Чекбокс карточки задачи | Активен карточный вид | Добавляет одну карточку в набор выбранных задач или убирает ее из набора. |
| Чекбокс строки таблицы | Активен табличный вид | Добавляет одну строку в набор выбранных задач или убирает ее из набора. |
| Чекбокс заголовка таблицы | Активен табличный вид; на текущей странице есть минимум одна видимая строка | Добавляет все видимые строки текущей страницы в набор выбранных задач или убирает их из набора. |

## 3. Плавающая панель массовых действий

| Часть панели | Состав |
|---|---|
| Счетчик результата | Текст `Результат:`; бейдж с числом выбранных задач. |
| Основные действия | `Изменить статус`; `Переместить в группу`. |
| Контекстные действия | `Назначить`; `Взять в работу`; `Завершить`; `Удалить из группы`. |
| Снятие выделения | Круглая кнопка с крестиком справа. |

| Состояние выбора | Видимость панели |
|---|---|
| Не выбрана ни одна задача | Плавающая панель скрыта. |
| Выбрана минимум одна задача | Плавающая панель видна в нижней центральной части блока списка задач. |

| Пространственное правило | Значение |
|---|---|
| Слой панели | Плавающий слой поверх блока списка задач в правой рабочей области. |
| Область привязки | Блок списка задач: карточный список или табличный список в правой части интерфейса. |
| Расположение | Нижняя центральная часть блока списка задач; центрирование выполняется относительно правой рабочей области, а не относительно всей ширины экрана. |
| Влияние на список задач | Панель находится поверх карточек или таблицы и не сдвигает список задач, пагинацию, панель инструментов, очереди и левую навигацию. |
| Граница с левой навигацией | Левая навигация не входит в область размещения панели и не участвует в расчете ее центрального положения. |
| Владелец видимости | Функциональный блок `Массовые действия` получает видимое состояние при наличии минимум одной выбранной задачи. |

## 4. Кнопки плавающей панели

| Кнопка | Условие показа | Структурная роль |
|---|---|---|
| `Изменить статус` | Выбрана минимум одна задача | Кнопка группового действия без реализованного видимого результата в текущем прототипе. |
| `Переместить в группу` | Выбрана минимум одна задача | Кнопка открытия дочернего UI `Перемещение в группу`. |
| `Назначить` | Среди выбранных задач есть задача из этапа `К назначению` | Контекстная кнопка группового действия без реализованного видимого результата в текущем прототипе. |
| `Взять в работу` | Среди выбранных задач есть задача из этапа `К исполнению` | Контекстная кнопка группового действия без реализованного видимого результата в текущем прототипе. |
| `Завершить` | Среди выбранных задач есть задача из этапа `В работе` | Контекстная кнопка группового действия без реализованного видимого результата в текущем прототипе. |
| `Удалить из группы` | Активная область относится к пользовательской группе, закладочной группе или сгруппированному значению | Контекстная кнопка удаления из пользовательской группы. |
| Крестик снятия выделения | Выбрана минимум одна задача | Кнопка очистки набора выбранных задач. |

## 5. Дочерний UI: модальное окно `Перемещение в группу`

| Часть модального окна | Состав |
|---|---|
| Хедер | Заголовок `Перемещение в группу`; кнопка закрытия. |
| Описание | Текст `Задачи будут объединены в новую кастомную группу.` |
| Поле | Подпись `Название новой группы`; поле ввода с плейсхолдером `Напр: Срочный ремонт БЦ`. |
| Футер | Кнопка `Отмена`; кнопка `Переместить`. |

| Пространственное правило | Значение |
|---|---|
| Слой окна | Модальный слой поверх интерфейса. |
| Точка открытия | Кнопка `Переместить в группу` в плавающей панели. |
| Влияние на плавающую панель | Плавающая панель остается состоянием функционального блока; модальное окно является дочерним UI этого блока. |
| Влияние на список задач | Модальное окно не меняет состав списка задач до подтверждения действия. |

## 6. Внешние условия и получатели результата

| Внешняя область | Роль для массовых действий | Структурная связь |
|---|---|---|
| Карточный и табличный список задач | В карточке и строке таблицы расположен чекбокс выбора одной задачи. Карточка или строка показывает выбранное состояние после выбора чекбокса. | Плавающая панель получает набор выбранных задач из карточек и строк списка задач. |
| Заголовок таблицы | В первой ячейке заголовка закрепленной колонки чекбоксов расположен чекбокс выбора всех видимых строк текущей страницы. | Чекбокс заголовка выбирает или снимает выбор со всех видимых строк текущей страницы; если выбрана минимум одна, но не все видимые строки текущей страницы, чекбокс получает состояние частичного выбора. |
| Фильтрация и поиск | Задают видимый набор задач. | Набор выбранных задач может включать задачи из текущего видимого набора. |
| Пагинация | Задает текущую страницу таблицы. | Чекбокс заголовка относится только к видимым строкам текущей страницы. |
| Настройки колонок | Меняют динамические колонки таблицы. | Чекбоксы выбора остаются доступными независимо от состава динамических колонок. |
| Очереди | Задают активный этап согласования для списка задач. | Состав контекстных кнопок панели зависит от этапов выбранных задач. |
| Левая навигация | Получает результат создания или удаления пользовательской группы. | Левая навигация является внешней областью результата группового действия. |
| Toast-уведомление | Получает текст результата действия. | Toast является внешней областью уведомления о групповом действии. |

Массовые действия не создают задачи, не меняют атрибуты задач, не меняют состав бизнес-доменов, не меняют поисковую строку и не меняют параметры фильтрации.
