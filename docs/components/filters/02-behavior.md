# Фильтрация: поведение

Файл описывает действия пользователя и точную реакцию интерфейса. Состав элементов см. в `01-structure.md`; правила состояния, счетчиков и отбора задач - в `03-data-states-rules.md`.

## 1. Основной пользовательский путь

Основной путь фильтрации: пользователь сужает текущий список задач и получает список только с подходящими задачами.

1. Пользователь находится в правой рабочей области со списком задач.
2. Пользователь нажимает кнопку `Фильтры`.
3. Справа открывается панель фильтрации.
4. Пользователь выбирает один или несколько параметров в панели.
5. Интерфейс показывает количество активных параметров в хедере панели и на кнопке `Фильтры`.
6. Пользователь нажимает `Применить`.
7. Панель закрывается.
8. Таблица или карточки показывают только задачи, которые проходят выбранные параметры.

Частные действия внутри этого пути описаны ниже таблицами. Правила прохождения задач через фильтры описаны в `03-data-states-rules.md`.

## 2. Открытие панели

Действие: пользователь нажимает кнопку `Фильтры` в верхней панели списка задач.

| Что происходит | Точный результат |
|---|---|
| Проверяется текущий контекст левой навигации | Если выбран конкретный бизнес-домен, в панели остаются общий блок и блок этого домена; блоки других доменов скрываются. Если выбран `Все задачи`, видны все доменные блоки. |
| Очищаются скрытые доменные фильтры | Для доменов, которые не отображаются в текущем контексте, значения возвращаются к состоянию по умолчанию. |
| Синхронизируется диапазон суммы | Поля `От` и `До`, бегунки, закрашенный участок шкалы и активность пресетов приводятся к текущему значению `sum`. |
| Счетчик активных параметров | Число в строке `Выполнено настроек: N` и число на индикаторе кнопки `Фильтры` становятся равными количеству активных параметров. |
| Открывается модальный слой | Overlay получает активное состояние, правая панель становится видимой. |

## 3. Закрытие панели без применения

Пользователь закрывает панель кликом по крестику в хедере или по overlay.

| Что меняется | Что не меняется |
|---|---|
| Overlay скрывается. Правая панель скрывается. | Значения, выбранные внутри панели, не сбрасываются. Кнопка `Применить` не нажимается автоматически. |

Если пользователь уже менял значения в панели, они остаются в активном состоянии фильтров и участвуют в счетчике. Таблица или карточки получают новый набор задач только после действия применения. Исключение: глобальный поиск применяет свое значение сразу при вводе.

## 4. Раскрытие мультивыбора

Действие: пользователь кликает по полю мультивыбора.

| Условие | Результат |
|---|---|
| Другой мультивыбор был открыт | Он закрывается. |
| Текущий мультивыбор был закрыт | Он раскрывается, список вариантов строится заново. |
| Текущий мультивыбор был открыт | Он закрывается. |
| Пользователь кликает по кнопке очистки или крестику тега | Раскрытие поля не выполняется, срабатывает соответствующее действие очистки. |

Правило построения вариантов описано в `03-data-states-rules.md`.

## 5. Выбор и снятие значения в мультивыборе

Действие: пользователь кликает по значению в выпадающем списке.

| Состояние значения до клика | Что меняется после клика |
|---|---|
| Значение не выбрано | Значение добавляется в массив выбранных значений параметра. В поле появляется тег выбранного значения. Кнопка очистки поля становится видимой. Число `N` в хедере панели и на индикаторе кнопки `Фильтры` становится равным количеству активных параметров. |
| Значение уже выбрано | Значение удаляется из массива выбранных значений параметра. Теги внутри поля перерисовываются. Если выбранных значений не осталось, возвращается плейсхолдер и скрывается кнопка очистки. Число `N` в хедере панели и на индикаторе кнопки `Фильтры` становится равным количеству активных параметров. |

Список задач не перерисовывается сразу после выбора или снятия значения. Для обновления списка пользователь нажимает `Применить`.

## 6. Очистка одного мультивыбора

Действие: пользователь нажимает кнопку очистки внутри конкретного мультивыбора.

| Что очищается | Что остается без изменений |
|---|---|
| Только массив выбранных значений этого параметра. Поле возвращает плейсхолдер. Кнопка очистки скрывается. Выбранные состояния вариантов снимаются. Число `N` в хедере панели и на кнопке `Фильтры` уменьшается, если этот параметр был активным. | Другие параметры фильтрации, глобальный поиск, диапазон суммы и видимость панели. |

Таблица или карточки получают новый набор задач после `Применить`.

## 7. Удаление выбранного тега

Действие: пользователь нажимает маленький крестик внутри тега выбранного значения.

| Условие | Результат |
|---|---|
| В параметре есть несколько выбранных значений | Удаляется только значение этого тега. Остальные выбранные значения остаются. |
| Это было последнее выбранное значение | Параметр становится пустым, поле показывает плейсхолдер, кнопка очистки скрывается. |

После удаления тега число `N` в хедере панели и на кнопке `Фильтры` уменьшается, если параметр стал пустым. Таблица или карточки получают новый набор задач после `Применить`.

## 8. Список вариантов мультивыбора

Список вариантов строится при раскрытии мультивыбора.

| Ситуация | Что видит пользователь |
|---|---|
| Есть доступные значения | Выпадающий список показывает значения, найденные в задачах текущего контекста. Выбранные значения имеют выбранное состояние. |
| Нет доступных значений | В списке показана строка `Нет доступных значений`. |

Текущий параметр при построении своих вариантов игнорируется, остальные активные параметры учитываются. Например, при открытии `Контрагент` выбранный `Контрагент` не сужает сам себя, но `Номер`, `Наименование`, `Тип документа` и другие активные параметры сужают список доступных контрагентов.

## 9. Ручной ввод суммы

Действие: пользователь вводит значение в поле `От` или `До`.

| Шаг обработки | Точный результат |
|---|---|
| Чтение ввода | Из текста берутся только цифры. Остальные символы игнорируются. |
| Округление | Значение округляется к шагу `10 000`. |
| Ограничение диапазоном | Значение не может стать меньше `0` или больше `20 000 000`. |
| Защита от пересечения | Если `От` становится больше `До`, изменяемая граница выравнивается так, чтобы не пересечь вторую. |
| Синхронизация UI | Соответствующий бегунок перемещается, закрашенный участок шкалы меняет положение и ширину. |
| Счетчик | Если заполнена хотя бы одна граница, параметр `Сумма, ₽` считается активным. |

Когда поле не в фокусе, значение отображается с разделением разрядов и знаком рубля. Пока поле в фокусе, значение показывается без знака рубля.

## 10. Изменение суммы ползунком

Действие: пользователь перемещает левый или правый бегунок.

| Бегунок | Что меняется |
|---|---|
| Левый | Обновляет нижнюю границу `От`. Если бегунок стоит на минимуме `0`, нижняя граница считается пустой. |
| Правый | Обновляет верхнюю границу `До`. Если бегунок стоит на максимуме `20 000 000`, верхняя граница считается пустой. |

Общие правила:

- левый бегунок не может оказаться правее правого;
- правый бегунок не может оказаться левее левого;
- поле `От` или `До` получает новое значение;
- закрашенный участок между бегунками меняет ширину и положение;
- число `N` в хедере панели и на кнопке `Фильтры` отражает текущее количество активных параметров.

Таблица или карточки получают новый набор задач после `Применить`.

## 11. Быстрые пресеты суммы

Действие: пользователь нажимает пресет под шкалой.

| Пресет | Значение `from` | Значение `to` |
|---|---:|---:|
| `до 500 тыс.` | пусто | `500 000` |
| `1 млн - 5 млн` | `1 000 000` | `5 000 000` |
| `свыше 10 млн` | `10 000 000` | пусто |

После выбора пресета поля `От` и `До`, бегунки, закрашенный участок шкалы и активное состояние пресета синхронизируются с выбранным диапазоном. Число `N` в хедере панели и на кнопке `Фильтры` отражает текущее количество активных параметров. Таблица или карточки получают новый набор задач после `Применить`.

## 12. Применение фильтров

Действие: пользователь нажимает `Применить`.

| Что происходит | Точный результат |
|---|---|
| Синхронизация суммы | Значения `От` и `До` читаются из полей, очищаются от лишних символов, приводятся к диапазону и записываются в `sum`. |
| Очистка скрытых доменов | Если выбран конкретный бизнес-домен, параметры других доменов возвращаются к значениям по умолчанию. |
| Счетчик | Число активных параметров отображается в хедере панели и на кнопке `Фильтры`. |
| Закрытие панели | Overlay и правая панель скрываются. |
| Список задач | Таблица или карточки перерисовываются по текущим фильтрам. |
| Левая панель | Счетчики задач в левой навигации пересчитываются с учетом фильтров. |

Один и тот же результат фильтрации используется для табличного и карточного отображения.

## 13. Полный сброс через футер

Действие: пользователь нажимает `Сбросить всё`.

| Область | Точный результат |
|---|---|
| Глобальный поиск | Поле `Поиск по задачам` очищается. |
| Мультивыборы | Массивы выбранных значений становятся пустыми. Теги удаляются. Плейсхолдеры возвращаются. Кнопки очистки скрываются. Выбранные состояния вариантов снимаются. |
| Диапазон суммы | `from` и `to` становятся пустыми. Поля очищаются. Бегунки возвращаются к полному диапазону. Активность пресетов снимается. |
| Счетчик | В хедере панели показывается `0`; индикатор на кнопке `Фильтры` скрывается, иконка фильтра возвращается. |
| Панель | Overlay и правая панель скрываются. |
| Список задач | Таблица или карточки перерисовываются без фильтров. |
| Левая панель | Счетчики задач пересчитываются без фильтров. |

## 14. Быстрый сброс на кнопке `Фильтры`

Быстрый сброс доступен только когда есть активные параметры и на кнопке `Фильтры` показан круглый индикатор с числом.

| Действие пользователя | Результат |
|---|---|
| Наведение на индикатор | Фон индикатора становится серым, число заменяется на крестик. |
| Клик по индикатору | Выполняется полный сброс из раздела 13. Клик не открывает панель фильтрации. |
| Клик по остальной части кнопки | Открывается панель фильтрации. |

## 15. Глобальный поиск

Действие: пользователь вводит текст в поле `Поиск по задачам`.

| Что меняется | Точный результат |
|---|---|
| Состояние фильтров | Значение `search` становится равным тексту поля без пробелов по краям. |
| Список задач | Таблица или карточки обновляются сразу после каждого изменения текста. |
| Левая панель | Счетчики задач пересчитываются сразу. |
| Панель фильтров | Если она была открыта, она остается открытой. |

Глобальный поиск не требует кнопки `Применить`.
