# Фильтрация: структура интерфейса

Файл описывает, где находится фильтрация и из каких видимых частей она состоит. Поведение действий описано в `02-behavior.md`; ключи данных, активные состояния и правила отбора задач - в `03-data-states-rules.md`.

## 1. Границы компонента

Фильтрация состоит из двух видимых зон:

| Зона | Где находится | Что делает |
|---|---|---|
| Кнопка `Фильтры` | Верхняя панель правой рабочей области со списком задач | Открывает правую панель фильтрации. При активных параметрах показывает счетчик и точку быстрого сброса. |
| Правая панель фильтрации | Открывается справа поверх интерфейса | Содержит параметры фильтрации, кнопку полного сброса и кнопку применения. |

Поле `Поиск по задачам` находится отдельно от панели фильтрации, в верхней панели списка задач. По правилам данных оно входит в общую фильтрацию; см. `03-data-states-rules.md`.

## 2. Кнопка `Фильтры`

Кнопка стоит в группе действий верхней панели правой рабочей области рядом с переключателем вида, скачиванием, созданием задачи и печатью.

| Состояние | Что видно на кнопке |
|---|---|
| Нет активных параметров | Иконка фильтра слева, текст `Фильтры`, скрытый служебный бейдж. |
| Есть активные параметры | Иконка фильтра скрыта. Вместо нее показан круглый индикатор 16 x 16 px с числом активных параметров. Текст `Фильтры` остается видимым. |
| Наведение на индикатор | Фон индикатора становится серым, число заменяется на крестик. |

Фон индикатора в обычном активном состоянии берется из основного цвета проекта. Быстрый сброс через индикатор описан в `02-behavior.md`.

## 3. Overlay

При открытии панели фильтрации поверх интерфейса появляется затемняющий overlay.

| Свойство | Описание |
|---|---|
| Расположение | За правой боковой панелью фильтрации. |
| Содержимое | Не содержит параметров, кнопок или собственных настроек. |
| Назначение | Создает модальный режим правой панели и закрывает панель при клике. |

## 4. Правая панель фильтрации

Панель открывается справа поверх основного интерфейса.

| Область панели | Состав |
|---|---|
| Хедер | Иконка фильтра в отдельном контейнере, заголовок `Фильтры`, строка `Выполнено настроек: N`, кнопка закрытия с крестиком. |
| Контент | Прокручиваемая вертикальная область с общим блоком и доменными блоками параметров. |
| Футер | Закрепленная нижняя область с кнопками `Сбросить всё` и `Применить`. |

`N` в хедере совпадает с числом в индикаторе кнопки `Фильтры`. Правило подсчета описано в `03-data-states-rules.md`.

## 5. Общий блок параметров

Общий блок показывается всегда.

| Параметр | Тип контрола | Плейсхолдер |
|---|---|---|
| `Номер` | Мультивыбор | `Выберите номер` |
| `Наименование` | Мультивыбор | `Выберите наименование` |

## 6. Доменные блоки параметров

Доменные блоки идут после общего блока. Условия их показа зависят от выбранного пункта левой навигации и описаны в `03-data-states-rules.md`.

| Блок | Бизнес-домен | Параметры внутри |
|---|---|---|
| `Строительные договоры` | `ecm` | `Тип документа`, `Контрагент`, `Сумма, ₽` |
| `Закупки ТМЦ` | `tracker` | `Бизнес-Юнит`, `SLA` |
| `Поручения СД` | `control` | `Объект`, `Критичность` |

## 7. Мультивыбор

Мультивыбор используется для всех параметров, кроме `Сумма, ₽`.

| Элемент мультивыбора | Когда виден | Назначение |
|---|---|---|
| Метка | Всегда | Называет параметр. |
| Кликабельное поле | Всегда | Открывает или закрывает выпадающий список. |
| Плейсхолдер | Пока нет выбранных значений | Подсказывает, что выбрать. |
| Тег выбранного значения | Когда выбрано минимум одно значение | Показывает первое выбранное значение. |
| Тег `+N` | Когда выбрано больше одного значения | Показывает количество дополнительных выбранных значений. |
| Кнопка очистки поля | Когда выбрано минимум одно значение | Очищает только этот параметр. |
| Стрелка раскрытия | Всегда | Показывает, что поле раскрывается. |
| Выпадающий список | Когда поле раскрыто | Показывает доступные значения или строку `Нет доступных значений`. |

Правила выбора, снятия выбора и построения доступных значений описаны в `02-behavior.md` и `03-data-states-rules.md`.

## 8. Диапазон суммы

Диапазон `Сумма, ₽` находится только в блоке `Строительные договоры`.

| Элемент | Значение в интерфейсе |
|---|---|
| Поле нижней границы | `От` |
| Поле верхней границы | `До` |
| Разделитель | `-` между полями |
| Шкала | Горизонтальная линия под полями |
| Бегунки | Левый для `От`, правый для `До` |
| Закрашенный участок | Показывает выбранный интервал между бегунками |
| Пресеты | `до 500 тыс.`, `1 млн - 5 млн`, `свыше 10 млн` |

| Настройка диапазона | Значение |
|---|---|
| Минимум | `0` |
| Максимум | `20 000 000` |
| Шаг | `10 000` |

## 9. Футер панели

Футер закреплен внизу панели.

| Кнопка | Расположение | Назначение |
|---|---|---|
| `Сбросить всё` | Слева в футере | Очищает все параметры фильтрации и закрывает панель. |
| `Применить` | Справа в футере | Применяет текущие параметры к списку задач и закрывает панель. |

Точное поведение кнопок описано в `02-behavior.md`.

## 10. Глобальный поиск

Поле `Поиск по задачам` находится в верхней панели правой рабочей области, над списком задач и вне правой панели фильтрации.

| Свойство | Описание |
|---|---|
| Видимый текст | `Поиск по задачам` |
| Входит в счетчик активных параметров | Да, если в поле есть текст. |
| Требует кнопки `Применить` | Нет. После каждого ввода таблица или карточки сразу показывают задачи, прошедшие поиск. |
| Очищается кнопкой `Сбросить всё` | Да. |
