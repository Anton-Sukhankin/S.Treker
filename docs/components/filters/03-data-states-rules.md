# Фильтрация: данные, состояния и правила

Файл описывает ключи данных, активные состояния, видимость доменных блоков и правила прохождения задач через фильтры. Структура интерфейса описана в `01-structure.md`; действия пользователя - в `02-behavior.md`.

## 1. Источник состояния

Текущие значения фильтров хранятся в общем состоянии активных фильтров.

| Ключ | Что означает | Видимый контрол в текущей панели |
|---|---|---|
| `search` | Глобальный поиск по задачам | Да, поле над списком задач |
| `id` | Номер задачи | Да, мультивыбор `Номер` |
| `title` | Наименование задачи | Да, мультивыбор `Наименование` |
| `docType` | Тип документа | Да, мультивыбор в блоке `Строительные договоры` |
| `contractor` | Контрагент | Да, мультивыбор в блоке `Строительные договоры` |
| `sum` | Сумма | Да, диапазон `Сумма, ₽` |
| `regDate` | Дата создания | Нет |
| `unit` | Бизнес-юнит | Да, мультивыбор в блоке `Закупки ТМЦ` |
| `sla` | SLA | Да, мультивыбор в блоке `Закупки ТМЦ` |
| `dueDate` | Срок | Нет |
| `object` | Объект | Да, мультивыбор в блоке `Поручения СД` |
| `criticality` | Критичность | Да, мультивыбор в блоке `Поручения СД` |

`regDate` и `dueDate` присутствуют в состоянии по умолчанию, но отдельных контролов для них в текущей панели фильтрации нет.

## 2. Видимые параметры

| Контекст | Видимые пользовательские параметры |
|---|---|
| Панель фильтрации в режиме `Все задачи` | 9 параметров: `Номер`, `Наименование`, `Тип документа`, `Контрагент`, `Сумма, ₽`, `Бизнес-Юнит`, `SLA`, `Объект`, `Критичность`. |
| Общая система фильтрации вместе с поиском | 10 параметров: 9 параметров панели плюс поле `Поиск по задачам`. |

Глобальный поиск находится вне панели, но участвует в тех же правилах отбора и подсчете активных параметров.

## 3. Общие и доменные параметры

| Группа | Когда доступна | Параметры и ключи |
|---|---|---|
| Общие параметры | Всегда | `Номер` -> `id`; `Наименование` -> `title`; `Поиск по задачам` -> `search` |
| `Строительные договоры` | В режиме `Все задачи` или при выбранном домене `ecm` | `Тип документа` -> `docType`; `Контрагент` -> `contractor`; `Сумма, ₽` -> `sum` |
| `Закупки ТМЦ` | В режиме `Все задачи` или при выбранном домене `tracker` | `Бизнес-Юнит` -> `unit`; `SLA` -> `sla` |
| `Поручения СД` | В режиме `Все задачи` или при выбранном домене `control` | `Объект` -> `object`; `Критичность` -> `criticality` |

## 4. Видимость доменных блоков

| Выбранный пункт левой панели | Что видно в панели фильтрации | Что происходит со скрытыми параметрами |
|---|---|---|
| `Все задачи` | Общий блок и все доменные блоки | Ничего не очищается из-за домена: все доменные фильтры могут участвовать в отборе. |
| Конкретный бизнес-домен | Общий блок и блок выбранного домена | Параметры других доменов очищаются при открытии панели и при `Применить`. |

Это правило исключает ситуацию, когда скрытый параметр другого домена незаметно сужает список задач.

## 5. Пустое и активное значение

| Тип значения | Пустое состояние | Активное состояние |
|---|---|---|
| Строка | `''`, `null` или `undefined` | Непустая строка |
| Массив мультивыбора | Пустой массив | В массиве есть хотя бы одно значение |
| Диапазон | `from: null` и `to: null` | Заполнена хотя бы одна граница |

Пустой параметр не влияет на список задач и не увеличивает счетчик активных параметров.

## 6. Подсчет активных параметров

Счетчик показывает количество активных параметров, а не количество выбранных значений.

| Ситуация | Добавка к счетчику |
|---|---:|
| В `Контрагент` выбраны три значения | `+1` |
| Выбраны `Контрагент` и `Тип документа` | `+2` |
| В сумме заполнены `От` и `До` | `+1` |
| Введен текст в `Поиск по задачам` | `+1` |
| Параметр пустой | `+0` |

Счетчик отображается в двух местах:

| Место | Что видно |
|---|---|
| Хедер панели | Строка `Выполнено настроек: N`. |
| Кнопка `Фильтры` | Если `N > 0`, иконка фильтра скрыта и вместо нее показан круглый индикатор с числом `N`. Если `N = 0`, индикатор скрыт и видна иконка фильтра. |

## 7. Базовое правило прохождения задачи

Задача остается в списке, если проходит все активные фильтры, применимые к текущему контексту.

| Условие | Результат |
|---|---|
| Фильтр пустой | Фильтр игнорируется. |
| Фильтр относится к другому домену, а выбран конкретный бизнес-домен | Фильтр не применяется к задачам текущего домена. |
| Фильтр активен и применим к контексту | Задача должна пройти правило этого фильтра. |
| Задача не проходит хотя бы один активный применимый фильтр | Задача скрывается из списка. |

## 8. Правила по типам фильтров

| Тип фильтра | Какие параметры используют | Правило прохождения задачи |
|---|---|---|
| Списочный фильтр | `Номер`, `Наименование`, `Тип документа`, `Контрагент`, `Бизнес-Юнит`, `SLA`, `Объект`, `Критичность` | Значение задачи должно точно совпасть хотя бы с одним выбранным значением. |
| Диапазон суммы | `Сумма, ₽` | У задачи должна быть сумма. Если `from` заполнен, сумма должна быть не меньше `from`. Если `to` заполнен, сумма должна быть не больше `to`. |
| Глобальный поиск | `Поиск по задачам` | Введенный текст ищется внутри строки, собранной из простых значений задачи. Регистр не учитывается. |

Если диапазон суммы активен, а у задачи нет значения суммы, задача не проходит фильтр суммы.

## 9. Построение вариантов в мультивыборе

Варианты мультивыбора строятся не из статического списка, а из задач текущего контекста.

| Часть контекста | Как влияет на варианты |
|---|---|
| Выбранный пункт левой панели | Ограничивает набор задач режимом `Все задачи`, конкретным бизнес-доменом, пользовательской группой или группировкой. |
| Активная очередь над списком задач | Ограничивает набор задач выбранной очередью. |
| Остальные активные фильтры | Сужают набор задач, из которого собираются варианты. |
| Текущий параметр | Игнорируется при построении собственных вариантов, чтобы выбранное значение не исключало само себя. |

Итог: настроенный фильтр по одному параметру уменьшает список доступных значений в других параметрах. Это взаимное сужение, а не жесткая последовательность сверху вниз.

## 10. Глобальный поиск

Глобальный поиск:

| Правило | Описание |
|---|---|
| Источник текста | Поле `Поиск по задачам` над списком задач. |
| Ключ состояния | `search`. |
| Применение | Сразу при вводе, без кнопки `Применить`. |
| Совместимость | Работает вместе с выбранной очередью, бизнес-доменом и другими фильтрами. |
| Сброс | Очищается при полном сбросе через `Сбросить всё` или быстрый индикатор на кнопке `Фильтры`. |

## 11. Влияние на список задач

После применения фильтров меняются только производные представления списка.

| Область | Что меняется |
|---|---|
| Таблица | Показывает только задачи, прошедшие текущие фильтры. |
| Карточки | Показывают тот же отфильтрованный набор задач, что и таблица. |
| Левая панель | Счетчики задач пересчитываются с учетом фильтров. |
| Очереди над списком | Счетчики очередей пересчитываются с учетом фильтров. |
| Пагинация | Работает по отфильтрованному набору. |
| Пустое состояние | Показывается, если после фильтрации нет подходящих задач. |

Фильтрация не меняет исходные данные задач, настройки колонок, выбранный режим отображения и состав бизнес-доменов.

## 12. Таблица и карточки

| Действие | Правило |
|---|---|
| Применить фильтры в таблице и перейти в карточки | Карточки получают тот же отфильтрованный набор задач. |
| Применить фильтры в карточках и перейти в таблицу | Таблица получает тот же отфильтрованный набор задач. |
| Переключить режим отображения | Активные фильтры не сбрасываются. |
