# Очереди: структура интерфейса

Этот файл описывает очередь как компонент интерфейса. Поведение описано в `02-behavior.md`, данные и правила индикации - в `03-data-states-rules.md`.

## 1. Где находится компонент

Очереди находятся в правой рабочей области, над основным списком задач.

Компонент расположен ниже заголовка текущего раздела и панели инструментов, но выше таблицы или карточного списка.

В интерфейсе очереди реализованы как горизонтальный сегмент-контрол.

## 2. Из чего состоит компонент

Компонент состоит из шести сегментов:

- `К исполнению`;
- `В работе`;
- `Обработаны мной`;
- `К назначению`;
- `Наблюдаю`;
- `Инициированы мной`.

Каждый сегмент является кнопкой переключения текущего набора отображаемых элементов списка.

## 3. Активный сегмент

В каждый момент активен один сегмент.

Активный сегмент визуально выделяется:

- светлой поверхностью;
- основным цветом текста;
- более заметной плотностью текста;
- легкой тенью.

По умолчанию активен сегмент `К исполнению`.

## 4. Счетчики

В текущем прототипе счетчики отображаются не у всех сегментов.

Счетчик есть у сегментов:

- `К исполнению`;
- `В работе`;
- `К назначению`.

Счетчик показывает количество элементов, которые попадают в соответствующую очередь с учетом текущего контекста интерфейса.

У сегментов `Обработаны мной`, `Наблюдаю` и `Инициированы мной` счетчик в разметке не отображается.

## 5. Индикация просрочки

Для сегмента `К исполнению` счетчик может получить красную индикацию.

Красная индикация означает, что внутри этой очереди есть хотя бы один просроченный элемент в текущем контексте.

Для остальных сегментов такая красная индикация в текущем прототипе не рассчитывается.

## 6. Когда компонент скрывается

Очереди отображаются в обычном режиме списка.

Если пользователь переходит в пользовательскую группу или другой кастомный список, сегмент-контрол очередей скрывается.

В кастомном списке пользователь видит уже собранный набор элементов без переключения по очередям.
