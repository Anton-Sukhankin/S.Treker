# Очереди: данные, состояния и правила

Этот файл описывает, какие значения используются для очередей, как считается активное состояние и как формируются счетчики. Структура компонента описана в `01-structure.md`, поведение - в `02-behavior.md`.

## 1. Источник данных

Список очередей формируется из `STATUS_MAP` в `src/data/statuses.js`.

На основе этого справочника в `src/data/task-model.js` создается `TASK_QUEUES`.

Каждая очередь имеет:

- `id`;
- `label`;
- `meaning`;
- `order`.

## 2. Значения очередей

В текущем прототипе используются такие очереди:

| Наименование | Системное значение |
|---|---|
| К исполнению | `todo` |
| В работе | `in_progress` |
| Обработаны мной | `done` |
| К назначению | `to_assign` |
| Наблюдаю | `review` |
| Инициированы мной | `initiated` |

## 3. Хранение в данных

В данных элемента списка используется поле `queueId`.

Для совместимости в части кода также встречается поле `statusId`, которое содержит то же значение очереди.

Это поле не является статусом объекта. Статус объекта описан отдельно в `docs/entities/task.md`.

## 4. Активное состояние

Активная очередь определяется активным сегментом в `.js-status-tabs`.

Сегмент считается активным, если у него есть класс `is-active`.

При клике по другому сегменту активное состояние переносится на выбранный сегмент.

## 5. Получение списка для очереди

При выборе сегмента интерфейс берет системное значение из `data-status`.

Далее системное значение переводится в человекочитаемую метку через обратную карту очередей.

После этого список перерисовывается для выбранной метки.

## 6. Правило счетчиков

Счетчик очереди показывает количество элементов, которые:

- относятся к этой очереди;
- находятся в текущем навигационном контексте;
- проходят активные фильтры.

Если подходящих элементов нет, счетчик показывает `0` и получает пониженную визуальную насыщенность.

## 7. Правило красной индикации

Красная индикация счетчика используется только для очереди `К исполнению`.

Счетчик становится красным, если:

- очередь имеет системное значение `todo`;
- количество подходящих элементов больше `0`;
- среди подходящих элементов есть хотя бы один просроченный.

Если просроченных элементов нет, счетчик остается в обычном состоянии.

## 8. Где лежит в проекте

- Справочник очередей: `src/data/statuses.js`
- Модель очередей: `src/data/task-model.js`
- Получение строк по очереди: `src/domain/task-selectors.js`
- Переключение сегментов: `src/features/tasks/tasks-view.js`
- Счетчики сегментов: `src/features/tasks/tasks-view.js`
- Массовые действия по очередям: `src/features/task-selection/floating-action-bar.js`
- Разметка сегментов: `index.html`
- Внешний вид сегментов и счетчиков: `src/styles/components/core-ui.css`

## 9. Когда обновлять документ

Обновлять, если меняется:

- список очередей;
- системные значения очередей;
- расположение сегмент-контрола;
- правила отображения счетчиков;
- правила красной индикации;
- связь очередей с фильтрами, бизнес-доменами или массовыми действиями.
