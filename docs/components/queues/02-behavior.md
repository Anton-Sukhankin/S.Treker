# Очереди: поведение

Этот файл описывает действия пользователя и реакцию интерфейса. Состав компонента описан в `01-structure.md`, правила данных и счетчиков - в `03-data-states-rules.md`.

## 1. Переключение очереди

Пользователь нажимает на сегмент очереди.

После нажатия:

- у всех сегментов снимается активное состояние;
- выбранный сегмент получает активное состояние;
- список задач перерисовывается под выбранную очередь;
- счетчики левой панели пересчитываются.

Переключение очереди не меняет выбранный бизнес-домен в левой панели.

## 2. Связь с бизнес-доменом

Очередь работает внутри текущего навигационного контекста.

Если выбран `Все задачи`, сегмент показывает элементы выбранной очереди по всем доступным бизнес-доменам.

Если выбран конкретный бизнес-домен, сегмент показывает элементы выбранной очереди только по этому бизнес-домену.

## 3. Связь с фильтрами

Фильтры применяются поверх выбранной очереди.

Если пользователь выбирает очередь, а затем применяет фильтры, список показывает только те элементы, которые одновременно:

- относятся к выбранной очереди;
- относятся к текущему бизнес-домену или режиму `Все задачи`;
- проходят активные фильтры.

Если фильтры уже активны, переключение очереди сохраняет фильтры и применяет их к новой очереди.

## 4. Связь с режимом отображения

Очередь не зависит от режима отображения списка.

Один и тот же результат выбранной очереди может быть показан:

- в таблице;
- в карточках.

Переключение между таблицей и карточками не сбрасывает активную очередь.

## 5. Счетчики после действий

Счетчики очередей пересчитываются после действий, которые меняют текущий набор отображаемых элементов:

- применение фильтров;
- сброс фильтров;
- переключение бизнес-домена;
- переключение пользовательской группы обратно в обычный список.

Если в очереди нет подходящих элементов, счетчик отображает `0`.

## 6. Массовые действия

Выбранная очередь влияет на набор массовых действий, которые доступны после выбора элементов списка.

В текущем прототипе массовые действия проверяют, из каких очередей выбраны элементы, и показывают подходящие кнопки действий.

Например:

- для элементов из `К исполнению` доступно действие перехода в работу;
- для элементов из `В работе` доступно действие завершения;
- для элементов из `К назначению` доступно действие назначения.
