# Компоненты интерфейса

Эта папка хранит спецификации компонентов интерфейса S-Tracker.

Старые короткие документы формата `interface.md` + `mechanics/` удалены. Они не являются источником истины и не должны использоваться как контекст для AI-агента.

## Текущая структура

Сейчас в целевом формате описаны:

- `filters/` - фильтрация задач;
- `queues/` - очереди над списком задач;
- `tasks/` - карточный и табличный список задач;
- `columns/` - настройки колонок таблицы.

## Фильтрация

Фильтрация описана в трех файлах:

- `filters/01-structure.md` - структура интерфейса фильтрации;
- `filters/02-behavior.md` - поведение пользователя и реакция интерфейса;
- `filters/03-data-states-rules.md` - данные, состояния и правила отбора.

## Очереди

Очереди описаны в трех файлах:

- `queues/01-structure.md` - расположение сегмент-контрола, состав сегментов и индикации;
- `queues/02-behavior.md` - переключение очередей и влияние на список;
- `queues/03-data-states-rules.md` - значения, счетчики, правила красной индикации и связь с кодом.

## Список задач

Список задач описан в трех файлах:

- `tasks/01-structure.md` - карточный вид, табличный вид, строки, карточки, закрепленные колонки, пагинация и пустое состояние;
- `tasks/02-behavior.md` - основной путь просмотра списка, переключение вида, выбор задач, пагинация, копирование email и реакция на фильтры;
- `tasks/03-data-states-rules.md` - источники данных, состояния, правила отбора, колонки, статусы, индикации и выбранные задачи.

## Настройки колонок

Настройки колонок описаны в трех файлах:

- `columns/01-structure.md` - точки входа, правая панель, список колонок, футер, связь с библиотекой и пресетами;
- `columns/02-behavior.md` - основной путь изменения колонок, черновик, применение, сброс, drag-and-drop и закрытие без применения;
- `columns/03-data-states-rules.md` - зоны реализации, состояния колонок, контексты, нормализация и правила применения к таблице.

## Как давать контекст AI-агенту

Не передавать всю папку `docs/components`.

Для конкретного компонента файлы передаются по порядку:

1. `01-structure.md`;
2. `02-behavior.md`;
3. `03-data-states-rules.md`.

Если задача касается только расположения и состава элементов, достаточно `01-structure.md`.

Если задача касается действий пользователя, добавить `02-behavior.md`.

Если задача касается данных, счетчиков, видимости или правил отбора, добавить `03-data-states-rules.md`.

## Правило для следующих компонентов

Новые спецификации компонентов создавать только в новом формате.

Не возвращать структуру `interface.md` + `mechanics/`.

Не создавать много коротких файлов с абстрактными описаниями. Один компонент должен описываться небольшим числом конкретных файлов, которые можно передавать агенту как рабочий контекст.
