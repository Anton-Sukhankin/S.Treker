# Настройки колонок: данные, состояния и правила

Этот файл фиксирует общее состояние родительского компонента `Настройки колонок`. Дочерние зоны `Пресеты колонок` и `Библиотека атрибутов` меняют черновик колонок родительского компонента; примененное состояние таблицы принадлежит родительскому компоненту.

## 1. Владельцы состояния

| Область | Состояние или значение | Правило видимого результата |
|---|---|---|
| `Настройки колонок` | Черновик колонок, примененное состояние, активный пресет, состояние раскрытия библиотеки | Открытая панель показывает черновик; таблица задач показывает только примененное состояние. |
| `Пресеты колонок` | Выбранный пункт пресета и сохраненные состояния колонок текущего контекста | Выбор пресета меняет черновик родительского компонента. |
| `Библиотека атрибутов` | Список доступных скрытых атрибутов и состояние кнопок добавления/возврата | Действия в библиотеке меняют черновик родительского компонента. |
| Табличный список задач | Видимый набор динамических колонок | Таблица получает новый набор динамических колонок только после `Применить`. |
| Карточный список задач | Карточное отображение задач | Карточный вид не получает состояние колонок из настроек колонок. |

## 2. Интерфейсные состояния настроек колонок

| Состояние | Значение | Видимый результат |
|---|---|---|
| Контекст настройки | `Все задачи`, бизнес-домен, пользовательская группа или другое представление | Определяет базовый набор настраиваемых колонок. |
| Базовое состояние | Минимальный набор отображаемых динамических колонок текущего контекста без библиотечных добавлений | Используется, если пользователь не применял настройки, выбрал `Базовое отображение` или нажал `По умолчанию`. |
| Примененное состояние | Последний набор колонок после `Применить` | Основная таблица показывает этот набор динамических колонок. |
| Черновик колонок | Временный набор динамических колонок внутри открытой панели: состав, порядок, видимость и признак добавления из библиотеки | Панель показывает черновик; таблица не меняется до `Применить`. |
| Активный пресет | Название выбранного пресета | Панель показывает колонки выбранного пресета. |
| Черновик активного пресета | Название пресета внутри открытой панели | Может отличаться от примененного пресета до `Применить`. |
| Библиотечные атрибуты в пресете | Есть или отсутствуют | При наличии таких атрибутов `Общие атрибуты` показывает их с кнопкой действия с иконкой стрелки возврата; раскрытая библиотека показывает у них кнопку действия с иконкой стрелки назад. |
| Библиотека атрибутов | Закрыта или открыта | При открытом состоянии доступны скрытые атрибуты текущего контекста. |

Главное правило: таблица читает примененное состояние, а открытая панель редактирует черновик. Владельцем черновика является родительский компонент `Настройки колонок`; список колонок, пресеты и библиотека атрибутов только читают или меняют этот черновик.

## 3. Правила контекста

| Контекст | Базовый набор колонок |
|---|---|
| `Все задачи` | Объединенный набор дефолтных колонок всех доступных бизнес-доменов. |
| Конкретный бизнес-домен | Дефолтные колонки выбранного бизнес-домена. |
| Пользовательская группа или другое представление вне списка доменов | Объединенный набор колонок, как для `Все задачи`. |

Один и тот же пресет или порядок колонок не переносится автоматически между разными контекстами.

Для одного и того же контекста базовое состояние остается одинаковым до пользовательской настройки колонок. Базовое состояние не включает атрибуты из библиотеки. Атрибуты из библиотеки появляются в черновике только после действия пользователя в дочерней зоне `Библиотека атрибутов` или после выбора сохраненного пресета, который содержит такие атрибуты.

## 4. Состояние одной колонки

| Свойство колонки | Правило |
|---|---|
| Идентификатор атрибута | Определяет, какую колонку описывает строка состояния. |
| Видимость | Включенная колонка попадает в таблицу после `Применить`; выключенная колонка не попадает. |
| Порядок | Порядок строк в черновике задает порядок динамических колонок после `Применить`. |
| Признак библиотечного атрибута | Колонка, добавленная из библиотеки, может быть возвращена в библиотеку. |

Фиксированные колонки чекбокса, номера и наименования не сохраняются в пользовательском состоянии. Они всегда добавляются таблицей отдельно и стоят первыми.

## 5. Базовое, черновое и примененное состояние

| Ситуация | Состояние панели | Состояние таблицы |
|---|---|---|
| Панель открыта без ранее примененных настроек | Черновик создан из базового состояния. | Таблица показывает базовое состояние. |
| Панель открыта после ранее примененных настроек | Черновик создан из примененного состояния. | Таблица показывает примененное состояние. |
| Черновик изменен без `Применить` | Панель показывает измененный черновик. | Таблица сохраняет прежнее примененное состояние. |
| Панель закрыта без `Применить` | Черновик отбрасывается. | Таблица сохраняет прежнее примененное состояние. |
| Нажато `Применить` | Черновик становится примененным состоянием. | Таблица показывает новый набор колонок. |
| Выбрано `По умолчанию` и нажато `Применить` | Примененное состояние возвращается к базовому набору. | Таблица показывает базовый набор колонок. |

## 6. Правила нормализации состояния

| Проверка | Итог |
|---|---|
| Состояние не является списком колонок | Используется базовое состояние. |
| У колонки нет понятного идентификатора атрибута | Колонка исключается из состояния. |
| Колонка относится к чекбоксу, номеру или наименованию | Колонка исключается из пользовательского состояния. |
| Колонка не входит в доступные колонки текущего контекста | Колонка исключается из состояния. |
| Колонка повторяется | Повтор исключается из состояния. |
| У колонки нет явного признака скрытия | Колонка считается видимой. |
| После нормализации состояние пустое | Таблица возвращается к базовому набору колонок. |

## 7. Правила активных колонок таблицы

| Шаг | Правило |
|---:|---|
| 1 | Берется базовый набор колонок текущего контекста. |
| 2 | К нему добавляются колонки из примененного состояния, если они были добавлены из библиотеки. |
| 3 | Фиксированные колонки чекбокса, номера и наименования ставятся первыми. |
| 4 | Остальные колонки сортируются по порядку в примененном состоянии. |
| 5 | Скрытые колонки исключаются. |
| 6 | Таблица показывает итоговый набор колонок. |

## 8. Правила библиотеки внутри настроек колонок

| Состояние библиотеки | Правило |
|---|---|
| Библиотека закрыта | Основная панель показывает только список `Общие атрибуты`. |
| Библиотека открыта | Показываются скрытые атрибуты текущего контекста. |
| Атрибут отсутствует в черновике | В библиотеке строка атрибута показывает кнопку действия с плюсом. |
| Атрибут добавлен из библиотеки | Атрибут входит в черновик как включенная колонка; в библиотеке строка атрибута показывает кнопку действия со стрелкой назад. |
| Атрибут возвращен через кнопку действия с иконкой стрелки назад | Атрибут отсутствует в черновике; в библиотеке строка атрибута показывает кнопку действия с иконкой плюса. |
| Библиотечные добавления сброшены | Все атрибуты, добавленные из библиотеки, отсутствуют в черновике; в библиотеке у этих атрибутов кнопки действия показывают иконку плюса. |

Библиотека не меняет основную таблицу до `Применить`.

## 9. Правила кнопки `Сохранить`

| Условие | Состояние кнопки |
|---|---|
| Черновик совпадает с выбранным пресетом или базой | `Сохранить` disabled. |
| Черновик отличается от выбранного пресета или базы | `Сохранить` active. |

Сравнение выполняется по набору колонок, их порядку и признаку видимости.
Библиотечные атрибуты участвуют в сравнении как часть набора колонок.

## 10. Контракты взаимодействия

| Связанный компонент | Роль связанного компонента | Условие | Результат в настройках колонок | Результат в связанном компоненте |
|---|---|---|---|---|
| Панель инструментов | Источник открытия настроек колонок | Пользователь нажал `Настроить колонки` | Панель настроек колонок открыта; черновик создан из примененного или базового состояния | Панель инструментов остается над списком задач. |
| Табличный список задач | Получатель примененного состояния | Пользователь нажал `Применить` | Черновик становится примененным состоянием | Таблица показывает примененный состав, порядок и видимость динамических колонок. |
| Табличный список задач | Получатель закрытия без применения | Пользователь закрыл панель без `Применить` | Черновик отбрасывается | Таблица сохраняет прежнее примененное состояние колонок. |
| Пресеты колонок | Дочерняя зона, меняющая черновик | Пользователь выбрал, сохранил или удалил пресет | Черновик и активный пресет получают состояние из дочерней зоны | Полная логика пресетов описана в `column-settings/presets`. |
| Библиотека атрибутов | Дочерняя зона, меняющая черновик | Пользователь добавил, вернул или сбросил библиотечные атрибуты | Черновик получает новый состав библиотечных атрибутов | Полная логика библиотеки описана в `column-settings/attribute-library`. |

## 11. Ограничения ответственности

| Область | Правило |
|---|---|
| Данные задач | Настройки колонок не изменяют значения задач. |
| Значения атрибутов | Настройки колонок не удаляют и не редактируют значения атрибутов. |
| Карточный вид | Карточный вид не зависит от настроек колонок. |
| Очереди | Настройки колонок не изменяют активную очередь и счетчики очередей. |
| Фильтры | Настройки колонок не изменяют параметры фильтрации. |
| Бизнес-домены | Настройки колонок не изменяют состав бизнес-доменов. |
