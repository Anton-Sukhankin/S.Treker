# Сущность: Статус задачи

Статус задачи — продуктовое состояние самой задачи. Статус отображается в списке задач как цветной тег.

Этапы согласования над списком задач не являются статусами задачи. Они описаны в [approval-stage.md](approval-stage.md).

## 1. Доступные статусы

| Наименование | Системное значение | Цвет тега |
|---|---|---|
| Новая | `new` | Циан |
| Назначена | `assigned` | Фиолетовый |
| В работе | `in_progress` | Синий |
| Назначение снято | `assignment_removed` | Серый |
| Завершена | `completed` | Зеленый |
| Отменена | `cancelled` | Темно-серый |
| Отклонена | `rejected` | Красный |

## 2. Правила видимого результата

| Условие | Видимый результат |
|---|---|
| У задачи есть системное значение статуса | В карточке и строке задачи показан тег с человекочитаемым названием статуса. |
| Статус задачи изменен в данных прототипа | Текст и цвет тега соответствуют новому системному значению. |
| Задача просрочена | Тег статуса не получает красную обводку; красным выделяется значение даты просрочки. |

## 3. Связанные компоненты

| Компонент | Как использует статус задачи |
|---|---|
| Список задач | Показывает тег статуса в карточном и табличном отображении. |
| Фильтрация | Может использовать статус как значение параметра, если параметр присутствует в текущем наборе. |
| Настройки колонок | Используют статус как колонку таблицы. |
| Группировка | Может группировать задачи по статусу. |

## 4. Ограничения

| Область | Правило |
|---|---|
| Этапы согласования | Статус задачи не определяет активный сегмент над списком задач. |
| Очереди | Очереди используют этап согласования, а не статус задачи. |
| История статусов | Entity-файл не описывает историю смены статусов. |
| Workflow | Entity-файл не задает правила перехода между статусами. |
