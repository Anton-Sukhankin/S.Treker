# Сущность: Этап согласования

Этап согласования — значение задачи, по которому задача попадает в один из сегментов над списком задач.

Этап согласования не является статусом задачи. Статус задачи описан в [task-status.md](task-status.md).

## 1. Этапы прототипа

| Название сегмента | Системное значение | Роль в интерфейсе |
|---|---|---|
| К исполнению | `todo` | Показывает задачи, ожидающие выполнения пользователем. |
| В работе | `in_progress` | Показывает задачи, находящиеся в работе. |
| Обработаны мной | `done` | Показывает задачи, обработанные пользователем. |
| К назначению | `to_assign` | Показывает задачи, ожидающие назначения. |
| Наблюдаю | `review` | Показывает задачи, где пользователь находится в наблюдателях. |
| Инициированы мной | `initiated` | Показывает задачи, созданные пользователем. |

## 2. Правила видимого результата

| Условие | Видимый результат |
|---|---|
| Сегмент выбран | Список задач показывает задачи выбранного этапа после выбранной области, поиска, фильтров и пагинации. |
| Задача относится к выбранному этапу | Задача может попасть в итоговый список, если проходит остальные условия отбора. |
| Задача относится к другому этапу | Задача не видна в списке активного сегмента. |
| Счетчик сегмента рассчитан | Сегмент показывает количество задач, которые проходят область, поиск и примененные фильтры для этого этапа. |

## 3. Связанные компоненты

| Компонент | Как использует этап согласования |
|---|---|
| Очереди | Показывают сегменты этапов и активный этап. |
| Список задач | Отбирает задачи активного этапа. |
| Панель инструментов | Размещает область сегментов под строкой инструментов. |
| Фильтрация | Примененные фильтры участвуют в счетчиках сегментов. |
| Левая навигация | Выбранная область задач ограничивает счетчики и список активного этапа. |

## 4. Ограничения

| Область | Правило |
|---|---|
| Статус задачи | Этап согласования не определяет текст и цвет тега статуса задачи. |
| Workflow | Entity-файл не описывает переход задачи между этапами. |
| Права пользователя | Entity-файл не описывает правила, по которым задача попадает пользователю в работу, наблюдение или инициированные. |
