# Сущности интерфейсных данных

`docs/entities` содержит минимальный словарь общих данных прототипа S-Tracker. Эти файлы нужны агенту только тогда, когда компонент использует данные, общие для нескольких областей интерфейса.

Папка не описывает серверную часть, хранение данных, полный набор сущностей реальной системы и внутреннюю реализацию текущего прототипа.

## Когда использовать

| Задача агента | Использовать `docs/entities` |
|---|---|
| Компонент показывает, отбирает или группирует задачи | Да, если нужно единое определение задачи, статуса, автора, бизнес-домена, этапа или атрибута. |
| Задача связывает несколько компонентов через общий набор данных | Да, вместе с контрактными разделами `03-data-states-rules.md` связанных компонентов. |
| Компонент использует только локальное состояние своей панели | Нет, правило остается в `03-data-states-rules.md` этого компонента. |
| Правка относится к расположению, кнопкам, панели, drawer, modal или toast | Нет, это зона `docs/components`. |
| Требуется техническая привязка к файлам кода | Нет, такая привязка не входит в entity-спецификацию. |

## Текущие сущности

| Сущность | Файл | Зачем нужна интерфейсу |
|---|---|---|
| Задача | [task.md](task.md) | Основной объект списка задач, фильтрации, очередей, группировки, пагинации и массовых действий. |
| Автор | [author.md](author.md) | Пользователь, указанный как автор задачи; отображается в строке и карточке задачи. |
| Статус задачи | [task-status.md](task-status.md) | Продуктовое состояние задачи, которое показывается цветным тегом. |
| Бизнес-домен | [business-domain.md](business-domain.md) | Область задач, которая ограничивает список, набор атрибутов, фильтры и настройки колонок. |
| Этап согласования | [approval-stage.md](approval-stage.md) | Значение, по которому задача попадает в сегмент над списком задач. |
| Атрибут задачи | [attribute.md](attribute.md) | Описание полей задачи, колонок таблицы, параметров фильтрации и библиотеки атрибутов. |
| Пользовательская группа | [user-group.md](user-group.md) | Сохраненный пользователем набор задач, который отображается в левой навигации и используется массовыми действиями. |
| Пресет колонок | [column-preset.md](column-preset.md) | Сохраненный состав колонок таблицы, включая атрибуты из библиотеки. |

## Правила папки

| Правило | Значение |
|---|---|
| Только данные прототипа | Описывать только значения, которые нужны текущему интерфейсу и спецификациям компонентов. |
| Без полной модели системы | Не добавлять сущности и поля, которые не участвуют в прототипе. |
| Без ссылок на код | Не указывать пути к исходному проекту, функции, структуру разметки, классы и техническую схему хранения. |
| Без интерфейсных сценариев | Действия пользователя, панели и пользовательские пути описываются в `docs/components` и `docs/user-flows`. |
| Без дублей компонентов | Entity-файл фиксирует смысл данных; компонентный `03` фиксирует, как компонент применяет эти данные. |
