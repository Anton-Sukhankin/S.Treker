# Сущность: Задача

Задача — основной объект данных, который отображается в списке задач и проходит через фильтрацию, очереди, группировку, пагинацию, настройки колонок и массовые действия.

Файл фиксирует только данные, которые нужны текущему интерфейсу прототипа.

## 1. Граница сущности

| Входит в сущность `Задача` | Не входит в сущность `Задача` |
|---|---|
| Идентификатор, наименование, бизнес-домен, этап согласования, статус задачи, автор, исполнитель, наблюдатели, даты, флаги индикации, значения атрибутов | Панель инструментов, фильтры, очереди, левая навигация, настройки колонок, пресеты, массовые действия, пагинация |

Компоненты читают значения задачи и формируют видимый результат. Компоненты не добавляют в сущность новые обязательные поля.

## 2. Данные задачи

| Данные | Тип значения | Где используется |
|---|---|---|
| Номер задачи | Уникальный текстовый идентификатор | Колонка `№`, карточка задачи, поиск, мультивыбор `Номер`. |
| Наименование | Текст | Колонка `Наименование`, карточка задачи, поиск, мультивыбор `Наименование`. |
| Бизнес-домен | Ссылка на [бизнес-домен](business-domain.md) | Левая навигация, объединенный список `Все задачи`, настройки колонок, фильтры, группировка. |
| Этап согласования | Ссылка на [этап согласования](approval-stage.md) | Сегменты над списком задач и счетчики сегментов. |
| Статус задачи | Ссылка на [статус задачи](task-status.md) | Тег статуса в карточном и табличном списке. |
| Автор | Ссылка на [автора](author.md) | Ячейка или область автора в карточке и строке задачи. |
| Исполнитель | Ссылка на пользователя | Колонка или значение `Исполнитель`, фильтрация и группировка по исполнителю. |
| Наблюдатели | Список ссылок на пользователей | Отбор задач для этапа `Наблюдаю`. |
| Дата создания | Дата | Отображение даты, поиск и фильтрация, индикация просрочки для строительных договоров. |
| Дата обновления | Дата | Служебное интерфейсное значение текущего прототипа; не имеет отдельного видимого поля в основном списке. |
| Флаг новой задачи | Логический признак | Зеленая индикация новой задачи в списке и группировках. |
| Флаг просрочки | Логический признак | Красная индикация строки или карточки и красное значение даты просрочки. |
| Значения атрибутов | Набор `атрибут -> значение` | Динамические колонки, фильтры, группировки, библиотека атрибутов. |

## 3. Правила видимого результата

| Условие в данных задачи | Видимый результат |
|---|---|
| Задача входит в выбранный бизнес-домен | Задача видна при выборе этого бизнес-домена, если проходит очередь, поиск, фильтры и текущую страницу. |
| Бизнес-домен задачи доступен текущей роли | Задача входит в область `Все задачи`. |
| Этап согласования задачи совпадает с активным сегментом | Задача входит в список активного сегмента. |
| Статус задачи задан | В списке задач показан тег статуса с человекочитаемым названием. |
| Автор задан | В списке задач показаны аватар автора, имя и email. |
| Значение атрибута есть у задачи | Таблица, фильтр или группировка используют это значение по ключу атрибута. |
| Значение атрибута отсутствует | В таблице для неприменимого или пустого значения показан пустой маркер. |
| Флаг новой задачи активен | Строка или карточка получает индикацию новой задачи. |
| Флаг просрочки активен | Строка или карточка получает индикацию просрочки; красным выделяется дата, определенная правилами списка задач. |

## 4. Связанные сущности

| Сущность | Связь с задачей |
|---|---|
| [Автор](author.md) | Задача содержит ссылку на пользователя, который отображается как автор. |
| [Статус задачи](task-status.md) | Задача содержит системное значение статуса. |
| [Бизнес-домен](business-domain.md) | Задача относится к одному бизнес-домену. |
| [Этап согласования](approval-stage.md) | Задача относится к одному этапу, который выбирается сегментом над списком. |
| [Атрибут задачи](attribute.md) | Задача содержит значения атрибутов, которые показываются в колонках и используются фильтрами. |

## 5. Ограничения

| Область | Правило |
|---|---|
| Сценарии пользователя | Действия пользователя с задачами описываются в компонентных спецификациях. |
| Полная модель реальной системы | Файл не перечисляет поля, которые не участвуют в текущем интерфейсе прототипа. |
| Серверная часть | Файл не описывает хранение, серверные методы, права доступа и интеграции. |
| Цвета и расположение компонентов | Визуальные правила находятся в компонентных спецификациях, если они являются продуктовым правилом интерфейса. |
