# Сущность: Задача

Этот файл описывает задачу как объект данных. Интерфейсные элементы, которые управляют списком задач, описываются отдельно в `docs/components`.

## Что это

Задача - основной объект S-Tracker.

В интерфейсе одна задача отображается как:

- одна строка в табличном списке;
- одна карточка в карточном списке.

Задача относится к бизнес-домену и имеет собственные атрибуты: номер, наименование, статус, исполнителя, автора, даты, сумму и доменные поля.

## Базовый состав данных

Объект задачи должен содержать данные, достаточные для отбора, отображения и связи с другими справочниками.

| Что хранит задача | Зачем нужно интерфейсу |
|---|---|
| Уникальный номер задачи | Показывается в списке и используется при поиске |
| Наименование | Показывается в карточке и в колонке `Наименование` |
| Бизнес-домен | Определяет, в каком разделе левой навигации находится задача |
| Очередь | Позволяет показать задачу в нужном сегменте над списком |
| Статус задачи | Определяет текст и цвет тега статуса |
| Автор | Связывает задачу с объектом пользователя, из которого берутся имя, email и аватар |
| Исполнитель | Нужен для отображения и возможной фильтрации |
| Даты | Используются в списке и для индикации просрочки |
| Значения доменных атрибутов | Заполняют колонки конкретного бизнес-домена и дополнительные атрибуты из библиотеки |
| Флаги индикации | Позволяют пометить задачу как новую или просроченную |

Значения доменных атрибутов могут храниться прямо в объекте задачи или отдельно в структуре вида `taskId` + `attributeKey` + `value`. Для интерфейса важно, чтобы по ключу атрибута можно было получить значение конкретной задачи.

## Атрибут `Статус`

У задачи есть атрибут **Статус**.

В данных он хранится как `taskStatusId`.

Колонка `Статус` в таблице и тег статуса в карточке показывают именно состояние самой задачи как объекта.

## Доступные статусы

| Наименование | Системное значение | Цвет тега |
|---|---|---|
| Новая | `new` | циан |
| Назначена | `assigned` | фиолетовый |
| В работе | `in_progress` | синий |
| Назначение снято | `assignment_removed` | серый |
| Завершена | `completed` | зеленый |
| Отменена | `cancelled` | темно-серый |
| Отклонена | `rejected` | красный |

## Визуальное отображение статуса

Статус задачи отображается в списке задач как цветной контрастный тег.

Правила тега:

- фон соответствует цвету статуса;
- текст всегда белый;
- текст тега содержит человекочитаемое наименование статуса;
- скругление тега статуса в списках задач - 16 пикселей;
- один и тот же статус выглядит одинаково в таблице и карточном списке.

Если задача просрочена, красная индикация применяется к значению срока, а не к тегу статуса.

## Что должно быть реализовано в коде

| Зона реализации | Что должна делать |
|---|---|
| Модель задачи | Хранить базовые поля задачи и ссылочные значения: бизнес-домен, очередь, статус, автора, исполнителя |
| Справочник статусов | Давать для каждого статуса системное значение, текст и цвет/класс тега |
| Справочник пользователей | Давать для автора имя, email и аватар |
| Механизм получения атрибутов | Возвращать значение задачи по ключу атрибута |
| Сборка строки списка | Превращать объект задачи и значения атрибутов в плоскую строку для карточки или таблицы |
| Отображение статуса | Показывать статус задачи как цветной тег с белым текстом |

## Текущая реализация в S-Tracker

Этот блок нужен для сопровождения текущего прототипа. Для генерации интерфейса главным является раздел `Что должно быть реализовано в коде`.

- Справочник статусов: `src/data/task-statuses.js`
- Поле задачи: `taskStatusId` в `src/data/task-model.js`
- Сборка строки для интерфейса: `src/domain/task-selectors.js`
- Отображение тега: `src/features/tasks/tasks-view.js`
- Стили цветов: `src/styles/components/core-ui.css`

## Когда обновлять документ

Обновлять, если меняется:

- список статусов;
- системные значения статусов;
- цвета тегов;
- правило отображения статуса в таблице или карточках;
- связь статуса с данными задачи.
