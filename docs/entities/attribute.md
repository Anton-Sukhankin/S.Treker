# Сущность: Атрибут задачи

Атрибут задачи — поле, которое может отображаться как колонка таблицы, участвовать в фильтрации, использоваться для группировки или добавляться из библиотеки атрибутов.

## 1. Типы атрибутов

| Тип атрибута | Правило интерфейса |
|---|---|
| Системный атрибут | Используется как базовая часть списка задач или служебная колонка. |
| Доменный атрибут | Доступен только для задач определенного бизнес-домена. |
| Атрибут библиотеки | Не виден в таблице по умолчанию; может быть добавлен через библиотеку атрибутов в настройках колонок. |
| Атрибут без значения у задачи | В таблице отображается как пустое значение. |

## 2. Значения атрибута

| Данные атрибута | Для чего нужно интерфейсу |
|---|---|
| Ключ атрибута | Связывает колонку, фильтр, группировку и значение задачи. |
| Название | Показывается в заголовке колонки, параметре фильтра или списке библиотеки атрибутов. |
| Тип значения | Определяет способ отображения и отбора: текст, число, дата, справочник, ссылка, управляющий элемент. |
| Справочник значений | Дает список вариантов для фильтра или отображения человекочитаемого значения. |
| Принадлежность бизнес-домену | Определяет, где атрибут доступен. |
| Видимость по умолчанию | Определяет, попадает ли атрибут в базовую таблицу. |
| Признак библиотеки | Определяет, доступен ли атрибут для добавления через библиотеку. |

## 3. Правила видимого результата

| Условие | Видимый результат |
|---|---|
| Атрибут видим по умолчанию для выбранного домена | Колонка атрибута входит в базовое отображение таблицы. |
| Атрибут находится в библиотеке | Атрибут виден в списке библиотеки атрибутов. |
| Атрибут добавлен из библиотеки в черновик настроек колонок | Общие атрибуты настроек колонок содержат этот атрибут с индикацией добавления. |
| Пользователь нажал `Применить` в настройках колонок | Добавленный атрибут становится колонкой таблицы. |
| Атрибут скрыт в настройках колонок | Колонка атрибута не видна в таблице после применения настроек. |
| Атрибут неприменим к бизнес-домену задачи в объединенной таблице | В ячейке показано пустое значение. |
| Атрибут используется для группировки | Левая навигация показывает значения атрибута как вложенную группировку. |
| Атрибут используется в фильтрации | Параметр фильтра проверяет задачи по значениям этого атрибута. |

## 4. Связанные компоненты

| Компонент | Как использует атрибут |
|---|---|
| Список задач | Показывает атрибуты как закрепленные или динамические колонки таблицы. |
| Фильтрация | Использует атрибуты как параметры отбора задач. |
| Настройки колонок | Используют атрибуты для изменения состава и порядка колонок. |
| Библиотека атрибутов | Показывает скрытые атрибуты, доступные для добавления в таблицу. |
| Пресеты колонок | Сохраняют состав, порядок, видимость и библиотечное происхождение атрибутов. |
| Левая навигация | Показывает группировки по значениям атрибутов. |

## 5. Ограничения

| Область | Правило |
|---|---|
| Полный справочник атрибутов реальной системы | Entity-файл не перечисляет атрибуты, которых нет в текущем прототипе. |
| Внутренняя логика компонентов | Entity-файл не описывает поведение таблицы, фильтра, селекта или drawer. |
| Настройка колонок | Правила черновика, применения и пресетов находятся в компоненте `Настройки колонок`. |
